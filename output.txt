Getting Started with zMenu
--------------------------

**zMenu** is the most powerful and flexible inventory (GUI) plugin for Minecraft servers. It allows you to create beautiful, interactive menus with an extensive configuration system that gives you complete control over every aspect of your inventories.

What is zMenu?[​](http://docs.groupez.dev/zmenu/getting-started#what-is-zmenu "Direct link to What is zMenu?")
--------------------------------------------------------------------------------------------------------------

zMenu is a premium-quality inventory plugin that enables server administrators to create custom GUIs (Graphical User Interfaces) for their Minecraft servers. Whether you need a simple server selector, a complex shop system, or an interactive quest menu, zMenu provides all the tools you need.

Key Features[​](http://docs.groupez.dev/zmenu/getting-started#key-features "Direct link to Key Features")
---------------------------------------------------------------------------------------------------------

### Highly Customizable Inventories[​](http://docs.groupez.dev/zmenu/getting-started#highly-customizable-inventories "Direct link to Highly Customizable Inventories")

*   Create inventories of any size (9 to 54 slots)
*   Multi-page support with automatic pagination
*   Custom titles with placeholder support
*   Fill items for empty slots
*   Matrix-based layout system for easy design

### Powerful Button System[​](http://docs.groupez.dev/zmenu/getting-started#powerful-button-system "Direct link to Powerful Button System")

*   **9 built-in button types**: NONE, INVENTORY, BACK, NEXT, PREVIOUS, HOME, JUMP, MAIN_MENU, SWITCH
*   Custom click actions for each mouse button
*   View and click requirements
*   Player head support with custom textures
*   Dynamic item updates

### Extensive Action System[​](http://docs.groupez.dev/zmenu/getting-started#extensive-action-system "Direct link to Extensive Action System")

*   **28+ action types** including messages, sounds, commands, teleportation, and more
*   Execute actions as player, console, or with OP permissions
*   Chain multiple actions together
*   Conditional actions with requirements

### Advanced Features[​](http://docs.groupez.dev/zmenu/getting-started#advanced-features "Direct link to Advanced Features")

*   **Pattern System**: Create reusable button templates
*   **Player Data**: Store and retrieve player-specific data
*   **Global Placeholders**: Define values used across all inventories
*   **PlaceholderAPI Support**: Full integration with PAPI
*   **MiniMessage Support**: Modern text formatting for Paper servers
*   **Anti-Dupe System**: Built-in protection against item duplication exploits
*   **Database Support**: MySQL, MariaDB, and SQLite for data persistence

### Developer Friendly[​](http://docs.groupez.dev/zmenu/getting-started#developer-friendly "Direct link to Developer Friendly")

*   Clean, well-documented API
*   Custom button type registration
*   Custom action type registration
*   Event system for inventory interactions

How It Works[​](http://docs.groupez.dev/zmenu/getting-started#how-it-works "Direct link to How It Works")
---------------------------------------------------------------------------------------------------------

zMenu uses YAML configuration files to define your inventories. Here's a simplified overview:

1.   **Inventories** are defined in the `inventories/` folder
2.   Each inventory contains **buttons** (items that players can interact with)
3.   Buttons can have **actions** that execute when clicked
4.   **Requirements** control who can see or click buttons
5.   **Patterns** allow you to reuse button configurations

### Simple Example[​](http://docs.groupez.dev/zmenu/getting-started#simple-example "Direct link to Simple Example")

`# inventories/my_menu.ymlname: "&6My First Menu"size: 27items:  welcome-button:    slot: 13    item:      material: DIAMOND      name: "&bWelcome!"      lore:        - "&7Click me to receive a message"    actions:      - type: message        messages:          - "&aHello, %player%!"          - "&7Thanks for using zMenu!"`

This creates a simple 27-slot inventory with a diamond in the center that sends a message when clicked.

Actions
-------

Actions are commands executed when a player interacts with a button. zMenu provides 28+ action types for various purposes.

Basic Usage[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#basic-usage "Direct link to Basic Usage")
---------------------------------------------------------------------------------------------------------------------

`items:  my-button:    slot: 0    item:      material: DIAMOND    actions:      - type: message        messages:          - "&aHello, %player%!"      - type: sound        sound: UI_BUTTON_CLICK`

Message Actions[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#message-actions "Direct link to Message Actions")
---------------------------------------------------------------------------------------------------------------------------------

*   message
*   messages (Alias)

`- type: message  messages:  - "&aLine 1"  - "&bLine 2"  minimessage: true  # Optional: Use MiniMessage format default: true`

`- type: messages          messages:          - "&aMultiple lines"          - "&bOf text"`

* * *

### message-to[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#message-to "Direct link to message-to")

Send a message to a specific player.

`- type: message-to  player: "Notch"  messages:    - "&aHello from %player%!"`

* * *

### broadcast[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#broadcast "Direct link to broadcast")

Send a message to all online players.

`- type: broadcast  messages:    - "&6[Announcement] &f%player% purchased a special item!"  minimessage: true  # Optional: Use MiniMessage format (default: true)`

* * *

### action-bar[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#action-bar "Direct link to action-bar")

*   action
*   actionbar (Alias)

`- type: action  message: "&aWelcome to the shop!"`

`- type: action_bar  message: "&aWelcome to the shop!"`

* * *

### title[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#title "Direct link to title")

`- type: title  title: "&6&lWelcome!"  subtitle: "&7Enjoy your stay"  start: 20      # Ticks  duration: 60   # Ticks  end: 20        # Ticks`

### change-title[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#change-title "Direct link to change-title")

Change the inventory title (requires PacketEvents plugin).

*   change-title
*   change-title-name (Alias)

`- type: change-title  inventory-name: "My New Inventory Title"`

`- type: change-title-name  inventory-name: "My New Inventory Title"`

**Options:**

| Option | Type | Required | Description |
| --- | --- | --- | --- |
| `inventory-name` | String | Yes | The new inventory title to display |

**Note:** This action requires the [PacketEvents](https://github.com/retrooper/packetevents) plugin to be installed on your server.

Available aliases: `change-title`, `change-title-name`.

* * *

### toast[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#toast "Direct link to toast")

`- type: "send toast"  icon: DIAMOND  message: "&aAchievement Unlocked!"  frame: TASK  # TASK, GOAL, or CHALLENGE`

Command Actions[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#command-actions "Direct link to Command Actions")
---------------------------------------------------------------------------------------------------------------------------------

### player-command[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#player-command "Direct link to player-command")

Execute a command as the player.

*   player-command
*   player command (Alias)

`- type: player-command  commands:    - "spawn"    - "kit starter"  command-in-chat: false  # Optional: execute via chat instead of dispatch (default: false)`

`- type: "player command"  commands:    - "spawn"    - "kit starter"  command-in-chat: false  # Optional: execute via chat instead of dispatch (default: false)`

**Options:**

| Option | Type | Required | Description |
| --- | --- | --- | --- |
| `commands` | List | Yes | List of commands to execute |
| `command-in-chat` | Boolean | No | Execute via chat instead of dispatch (default: false) |

**Note:** This action supports multiple aliases for flexibility.

Available aliases: `player-command`, `player_command`, `player_commands`, `player command`, `player commands`, `player-commands`.

* * *

### player-command-as-op[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#player-command-as-op "Direct link to player-command-as-op")

Execute a command as the player with OP permissions.

*   player-commands-as-op
*   player command as op (Alias)

`- type: player-command-as-op  commands:    - "gamemode creative"`

`- type: "player command as op"  commands:    - "gamemode creative"`

**Options:**

| Option | Type | Required | Description |
| --- | --- | --- | --- |
| `commands` | List | Yes | Commands to execute as OP |

**Note:** Remember to enable this action in your configuration.

Available aliases: `player-command-as-op`, `player commands as op`, `player command as op`, `player_commands_as_op`, and `player_command_as_op`.

Use with caution! This gives the player temporary OP-level access.

You need to enable it in the configuration file:

`enable-player-commands-as-op-action: true`

**Options:**

### random-player-command[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#random-player-command "Direct link to random-player-command")

Execute a random selection of commands as the player from a list. The commands are selected **without replacement** (no duplicates in a single execution).

`- type: random_player_command  commands:    - "say I got option 1!"    - "say I got option 2!"    - "say I got option 3!"    - "say I got option 4!"  amount: 1               # Number of commands to randomly execute (default: 1)  command-in-chat: false   # Optional: execute via chat instead of dispatch (default: false)`

**Options:**

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `commands` | List | Required | List of commands to randomly select from |
| `amount` | Integer | `1` | Number of random commands to execute |
| `command-in-chat` | Boolean | `false` | If `true`, sends via `player.chat()`; if `false`, dispatches as command |

**Aliases:**`random_player_command`, `random_player_commands`, `random-player-command`, `random-player-commands`

* * *

### random-console-command[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#random-console-command "Direct link to random-console-command")

Execute a random selection of console commands from a list. The commands are selected **without replacement** (no duplicates in a single execution). Supports the `%player%` placeholder.

`- type: random_console_command  commands:    - "give %player% diamond 1"    - "give %player% emerald 1"    - "give %player% gold_ingot 1"    - "give %player% iron_ingot 1"  amount: 2    # Execute 2 random commands from the list`

**Options:**

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `commands` | List | Required | List of console commands to randomly select from |
| `amount` | Integer | `1` | Number of random commands to execute |

**Aliases:**`random_console_command`, `random_console_commands`, `random-console-command`, `random-console-commands`, `random-command`, `random-commands`

| `webhook` | String | Yes | Webhook URL for the Discord server. |
| `content` | String | Yes | Message text or content to send. |
| `username` | String | No | Optional name displayed for the sender. |
| `avatar_url` | URL | No | Optional avatar for the sender. |

**Note:** Supports webhook-based interactions with Discord.

Available aliases: `discord`, `send-discord`, `discord webhook`, `discordwebhook`.

`- type: "discord component"  webhook: "https://discord.com/api/webhooks/..."  embeds:  - title: "New Purchase"  description: "%player% bought an item"  color: "#00FF00"`

Available aliases: `discord_component`, `discord component`, `discord webhook component`, `discordwebhookcomponent`.

Action Order[​](http://docs.groupez.dev/zmenu/configurations/buttons/actions#action-order "Direct link to Action Order")
------------------------------------------------------------------------------------------------------------------------

Actions execute in the order they're listed. If you need to close the inventory after a teleport, put `close` before `teleport`:

`actions:  - type: close  - type: teleport    world: "world"    x: 0    y: 100    z: 0`

Button Configuration
--------------------

Buttons are the interactive elements in your inventories. This page documents all available button configuration options.

Basic Structure[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#basic-structure "Direct link to Basic Structure")
--------------------------------------------------------------------------------------------------------------------------------

`items:  my-button:    slot: 0    item:      material: DIAMOND      name: "&bMy Button"    actions:      - type: message        messages:          - "&aYou clicked the button!"`

Configuration Options[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#configuration-options "Direct link to Configuration Options")
--------------------------------------------------------------------------------------------------------------------------------------------------

### slot[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#slot "Direct link to slot")

The position of the button in the inventory. Slots are numbered from 0.

`items:  button:    slot: 13`

![Image 1: Description](https://docs.groupez.dev/img/slots.png)

* * *

### slots[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#slots "Direct link to slots")

Use multiple slots for the same button.

`items:  border:    slots:      - 0      - 1      - 2      - 8    item:      material: BLACK_STAINED_GLASS_PANE`

**Range Syntax:**

`items:  row:    slots:      - 0-8      # Slots 0 through 8      - 45-53    # Slots 45 through 53`

* * *

### type[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#type "Direct link to type")

The button type determines special behavior. Default is `NONE`.

`items:  back:    slot: 45    type: BACK    item:      material: ARROW      name: "&c&lBack"`

**Available Types:**

| Type | Description |
| --- | --- |
| `NONE` | Default button with no special behavior |
| `INVENTORY` | Opens another inventory |
| `BACK` | Returns to the previous inventory |
| `HOME` | Returns to the first inventory in history |
| `NEXT` | Goes to the next page |
| `PREVIOUS` | Goes to the previous page |
| `JUMP` | Jumps to a specific page |
| `MAIN_MENU` | Opens the main menu |
| `SWITCH` | Displays different items based on conditions |
| `PAGINATION` | Display paginated content (zMenu+ only) |
| `DYNAMIC_PAGINATION` | Dynamic pagination with variable content (zMenu+ only) |
| `INPUT` | Input button for player input (zMenu+ only) |

See [Button Types](https://docs.groupez.dev/zmenu/configurations/buttons/types/none) for detailed documentation on each type.

* * *

### item[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#item "Direct link to item")

The visual appearance of the button. See [Item Configuration](https://docs.groupez.dev/zmenu/configurations/items/item) for all options.

`items:  button:    slot: 0    item:      material: DIAMOND      name: "&b&lDiamond"      lore:        - "&7A shiny diamond"`

* * *

### actions[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#actions "Direct link to actions")

Actions executed when the button is clicked.

`items:  button:    slot: 0    item:      material: EMERALD    actions:      - type: message        messages:          - "&aYou clicked!"      - type: sound        sound: UI_BUTTON_CLICK`

See [Actions](https://docs.groupez.dev/zmenu/configurations/buttons/actions) for all available action types.

* * *

### sound[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#sound "Direct link to sound")

Play a sound when the button is clicked (shorthand).

`items:  button:    slot: 0    sound: UI_BUTTON_CLICK    item:      material: DIAMOND`

For more control, use the sound action instead:

`actions:  - type: sound    sound: UI_BUTTON_CLICK    pitch: 1.5    volume: 0.5`

* * *

### messages[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#messages "Direct link to messages")

Send messages when clicked (shorthand).

`items:  button:    slot: 0    messages:      - "&aHello!"      - "&7Welcome to the server."    item:      material: BOOK`

* * *

### close-inventory[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#close-inventory "Direct link to close-inventory")

Close the inventory when the button is clicked.

`items:  close:    slot: 49    close-inventory: true    item:      material: BARRIER      name: "&c&lClose"`

* * *

### refresh-on-click[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#refresh-on-click "Direct link to refresh-on-click")

Refresh the entire inventory after clicking.

`items:  refresh:    slot: 0    refresh-on-click: true    item:      material: COMPASS      name: "&e&lRefresh"`

* * *

### update-on-click[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#update-on-click "Direct link to update-on-click")

Update only this button after clicking.

`items:  toggle:    slot: 0    update-on-click: true    item:      material: LEVER      name: "&e&lToggle"`

* * *

### update[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#update "Direct link to update")

Enable automatic updates for this button (refreshes placeholders and dynamic content).

`items:  dynamic-info:    slot: 0    update: true    item:      material: PAPER      name: "&e&lBalance: &a$%vault_eco_balance%"`

* * *

### update-master-button[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#update-master-button "Direct link to update-master-button")

Update the master button when this button is updated. Useful for SWITCH buttons.

`items:  child-button:    slot: 0    update-master-button: true    item:      material: DIAMOND`

* * *

### use-cache[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#use-cache "Direct link to use-cache")

Enable caching for this button to improve performance.

`items:  cached-button:    slot: 0    use-cache: true    item:      material: DIAMOND      name: "&bCached Item"`

When enabled, the button's item will be cached and reused instead of being rebuilt each time.

* * *

### is-permanent[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#is-permanent "Direct link to is-permanent")

Display this button on all pages of a paginated inventory.

`items:  navigation:    slot: 49    is-permanent: true    item:      material: ARROW      name: "&7Navigation"`

* * *

### page[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#page "Direct link to page")

Display button only on a specific page.

`items:  page1-only:    slot: 0    page: 1    item:      material: DIAMOND      name: "&bPage 1 Item"`

**Range format:**

`items:  pages-1-to-3:    slot: 0    page: 1-3    item:      material: EMERALD      name: "&aPages 1-3"`

* * *

### else[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#else "Direct link to else")

Define an alternative button when a view requirement isn't met.

`items:  vip-button:    slot: 0    view-requirement:      requirements:        - type: permission          permission: "vip.access"    item:      material: DIAMOND_BLOCK      name: "&b&lVIP Content"    else:      item:        material: COAL_BLOCK        name: "&7&lLocked"        lore:          - "&cRequires VIP rank"`

* * *

### open-link[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#open-link "Direct link to open-link")

This configuration is deprecated. Use MiniMessage format to open links instead:

`item:  name: "<click:open_url:'https://discord.gg/myserver'>&9&lClick to join Discord</click>"`

Open a URL or prompt the player to join a Discord server.

`items:  discord:    slot: 0    open-link: "https://discord.gg/myserver"    item:      material: PLAYER_HEAD      url: "discord_head_texture"      name: "&9&lDiscord"      lore:        - "&7Click to join our Discord!"`

* * *

### permission[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#permission "Direct link to permission")

Require a permission to see the button. If the player doesn't have the permission, the button will be hidden.

`items:  admin-button:    slot: 0    permission: "server.admin"    item:      material: COMMAND_BLOCK      name: "&c&lAdmin Panel"`

You can also use a list of permissions (all required):

`items:  special-button:    slot: 0    permission:      - "server.vip"      - "server.premium"    item:      material: DIAMOND_BLOCK      name: "&b&lVIP Premium"`

* * *

### or-permission[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#or-permission "Direct link to or-permission")

Require at least one of the listed permissions to see the button.

*   or-permission
*   orPermissions (Aliase)
*   or-permissions (Aliase)

`# YAML example for or-permissionitems:  staff-button:    slot: 0    or-permission:      - "server.admin"      - "server.moderator"      - "server.helper"    item:      material: BOOK      name: "&6&lStaff Menu"`

`# YAML example for orPermissionsitems:  staff-button:    slot: 0    orPermissions:      - "server.admin"      - "server.moderator"      - "server.helper"    item:      material: BOOK      name: "&6&lStaff Menu"`

`# YAML example for or-permissionsitems:  staff-button:    slot: 0    or-permissions:      - "server.admin"      - "server.moderator"      - "server.helper"    item:      material: BOOK      name: "&6&lStaff Menu"`

* * *

### placeholder[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#placeholder "Direct link to placeholder")

Check a placeholder value to determine if the button should be displayed.

*   placeholder
*   placeHolder (Aliase)

`# YAML example for placeholderitems:  level-button:    slot: 0    placeholder: "%player_level%"    action: ">="    value: "10"    item:      material: EXPERIENCE_BOTTLE      name: "&a&lLevel 10+ Reward"`

`# YAML example for placeHolderitems:  level-button:    slot: 0    placeHolder: "%player_level%"    action: ">="    value: "10"    item:      material: EXPERIENCE_BOTTLE      name: "&a&lLevel 10+ Reward"`

**Available Actions:**

| Action | Description |
| --- | --- |
| `==` | Equals |
| `!=` | Not equals |
| `>=` | Greater than or equals |
| `>` | Greater than |
| `<=` | Less than or equals |
| `<` | Less than |
| `equals_string` | String equals |
| `equalsIgnoreCase` | String equals (ignore case) |

* * *

### commands[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#commands "Direct link to commands")

It is recommended to use [Actions](https://docs.groupez.dev/zmenu/configurations/buttons/actions) instead of `commands` for more flexibility and features.

Execute commands as the player when the button is clicked.

*   commands
*   playerCommands (Aliase)
*   player-commands (Aliase)

`# YAML example for commandsitems:  warp-button:    slot: 0    commands:      - "warp spawn"      - "say I teleported!"    item:      material: ENDER_PEARL      name: "&a&lTeleport to Spawn"`

`# YAML example for playerCommandsitems:  warp-button:    slot: 0    playerCommands:      - "warp spawn"      - "say I teleported!"    item:      material: ENDER_PEARL      name: "&a&lTeleport to Spawn"`

`# YAML example for player-commandsitems:  warp-button:    slot: 0    player-commands:      - "warp spawn"      - "say I teleported!"    item:      material: ENDER_PEARL      name: "&a&lTeleport to Spawn"`

You can also specify commands for specific click types:

*   rightCommands
*   right-commands (Aliase)
*   rightPlayerCommands (Aliase)
*   right-player-commands (Aliase)

`# YAML example for rightCommandsitems:  multi-command:    slot: 0    rightCommands:      - "warp hub"    item:      material: COMPASS`

`# YAML example for right-commandsitems:  multi-command:    slot: 0    right-commands:      - "warp hub"    item:      material: COMPASS`

`# YAML example for rightPlayerCommandsitems:  multi-command:    slot: 0    rightPlayerCommands:      - "warp hub"    item:      material: COMPASS`

`# YAML example for right-player-commandsitems:  multi-command:    slot: 0    right-player-commands:      - "warp hub"    item:      material: COMPASS`

*   leftCommands
*   left-commands (Aliase)
*   leftPlayerCommands (Aliase)
*   left-player-commands (Aliase)

stop-on-empty[​](http://docs.groupez.dev/zmenu/configurations/buttons/button#stop-on-empty "Direct link to stop-on-empty")
--------------------------------------------------------------------------------------------------------------------------

The `stop-on-empty` flag determines whether default actions are added when no actions are specified for a button.

*   **Enabled (`true`)**: Only the actions you define are used. Default actions are not added.
*   **Disabled (`false`)**: Default actions are automatically added if no actions are explicitly defined.

To control this behavior for your buttons:

*   Use `stop-on-empty: true` to load only your own actions.
*   Use `stop-on-empty: false` to ensure default actions are added when no actions are present.

Configuration Informations
--------------------------

This page provides essential information about how to configure zMenu and understand its configuration philosophy.

Understanding YAML[​](http://docs.groupez.dev/zmenu/configurations/informations#understanding-yaml "Direct link to Understanding YAML")
---------------------------------------------------------------------------------------------------------------------------------------

zMenu uses YAML (YAML Ain't Markup Language) for all its configuration files. If you're new to YAML, here are some important things to know:

### Basic YAML Rules[​](http://docs.groupez.dev/zmenu/configurations/informations#basic-yaml-rules "Direct link to Basic YAML Rules")

1.   **Indentation matters** - Use spaces (not tabs) for indentation
2.   **Consistent spacing** - Use 2 spaces per indentation level
3.   **Case sensitive** - `Material` is different from `material`
4.   **Colons need spaces** - `key: value` not `key:value`

### Example Structure[​](http://docs.groupez.dev/zmenu/configurations/informations#example-structure "Direct link to Example Structure")

`# This is a commentinventory-name: "My Inventory"    # String valuesize: 54                          # Number valueenabled: true                     # Boolean valueitems:                            # Start of a section  my-button:                      # Button name (key)    slot: 0                       # Property of my-button    item:                         # Nested section      material: DIAMOND      name: "&bDiamond"`

### Useful Tools[​](http://docs.groupez.dev/zmenu/configurations/informations#useful-tools "Direct link to Useful Tools")

*   **YAML Validator**: [YAML Lint](http://www.yamllint.com/) - Check your YAML syntax
*   **VS Code Extension**: YAML by Red Hat - Syntax highlighting and validation
*   **Notepad++**: Lightweight editor with YAML support

Configuration Philosophy[​](http://docs.groupez.dev/zmenu/configurations/informations#configuration-philosophy "Direct link to Configuration Philosophy")
---------------------------------------------------------------------------------------------------------------------------------------------------------

zMenu follows a modular configuration approach:

### File Organization[​](http://docs.groupez.dev/zmenu/configurations/informations#file-organization "Direct link to File Organization")

`plugins/zMenu/├── config.yml                 # Global settings├── global-placeholders.yml    # Shared values├── commands/                  # Command definitions│   └── commands.yml├── inventories/               # Your menus│   ├── menu1.yml│   └── subfolder/│       └── menu2.yml├── patterns/                  # Reusable templates├── items/                     # Reusable items└── actions_patterns/          # Default actions`

### Key Principles[​](http://docs.groupez.dev/zmenu/configurations/informations#key-principles "Direct link to Key Principles")

1.   **One inventory per file** - Each YAML file in `inventories/` defines one inventory
2.   **Subfolders supported** - Organize inventories in subfolders
3.   **Reusability** - Use patterns and items to avoid repetition
4.   **Hot reload** - Use `/zm reload` to apply changes without restart

Color Codes[​](http://docs.groupez.dev/zmenu/configurations/informations#color-codes "Direct link to Color Codes")
------------------------------------------------------------------------------------------------------------------

zMenu supports multiple color code formats:

### Legacy Color Codes[​](http://docs.groupez.dev/zmenu/configurations/informations#legacy-color-codes "Direct link to Legacy Color Codes")

Using the `&` symbol:

| Code | Color | Code | Color |
| --- | --- | --- | --- |
| `&0` | Black | `&8` | Dark Gray |
| `&1` | Dark Blue | `&9` | Blue |
| `&2` | Dark Green | `&a` | Green |
| `&3` | Dark Aqua | `&b` | Aqua |
| `&4` | Dark Red | `&c` | Red |
| `&5` | Dark Purple | `&d` | Light Purple |
| `&6` | Gold | `&e` | Yellow |
| `&7` | Gray | `&f` | White |

### Formatting Codes[​](http://docs.groupez.dev/zmenu/configurations/informations#formatting-codes "Direct link to Formatting Codes")

| Code | Effect |
| --- | --- |
| `&l` | **Bold** |
| `&o` | _Italic_ |
| `&n` | Underline |
| `&m` | ~~Strikethrough~~ |
| `&k` | Obfuscated |
| `&r` | Reset |

### Hex Colors[​](http://docs.groupez.dev/zmenu/configurations/informations#hex-colors "Direct link to Hex Colors")

`name: "&#FF5555This is red &#55FF55and this is green"`

### MiniMessage (Paper/Purpur)[​](http://docs.groupez.dev/zmenu/configurations/informations#minimessage-paperpurpur "Direct link to MiniMessage (Paper/Purpur)")

If you're using Paper, Purpur, or Pufferfish, you can enable MiniMessage format in `config.yml`:

`enable-mini-message-format: true`

Then use MiniMessage syntax:

Placeholders[​](http://docs.groupez.dev/zmenu/configurations/informations#placeholders "Direct link to Placeholders")
---------------------------------------------------------------------------------------------------------------------

zMenu supports placeholders throughout your configurations:

### PlaceholderAPI[​](http://docs.groupez.dev/zmenu/configurations/informations#placeholderapi "Direct link to PlaceholderAPI")

Any PlaceholderAPI placeholder works in zMenu:

`name: "&6%player_name%'s Profile"lore:  - "&7Balance: &a$%vault_eco_balance%"  - "&7Level: &e%player_level%"`

### zMenu Built-in Placeholders[​](http://docs.groupez.dev/zmenu/configurations/informations#zmenu-built-in-placeholders "Direct link to zMenu Built-in Placeholders")

| Placeholder | Description |
| --- | --- |
| `%player%` | Player name |
| `%page%` | Current page number |
| `%maxPage%` or `%max-page%` | Total pages |
| `%zmenu_player_page%` | Current page |
| `%zmenu_player_max_page%` | Maximum page |

See the [Placeholders page](https://docs.groupez.dev/zmenu/configurations/placeholders) for the complete list.

Reloading Configurations[​](http://docs.groupez.dev/zmenu/configurations/informations#reloading-configurations "Direct link to Reloading Configurations")
---------------------------------------------------------------------------------------------------------------------------------------------------------

After making changes, reload your configurations:

`/zm reload                    # Reload everything/zm reload config             # Reload config.yml only/zm reload inventory [name]   # Reload specific inventory/zm reload command [name]     # Reload specific command`

Use `/zm reload inventory <name>` when testing changes to a specific inventory - it's faster than reloading everything.

Common Mistakes[​](http://docs.groupez.dev/zmenu/configurations/informations#common-mistakes "Direct link to Common Mistakes")
------------------------------------------------------------------------------------------------------------------------------

### 1. Incorrect Indentation[​](http://docs.groupez.dev/zmenu/configurations/informations#1-incorrect-indentation "Direct link to 1. Incorrect Indentation")

`# Wrongitems:my-button:  slot: 0# Correctitems:  my-button:    slot: 0`

### 2. Missing Quotes for Special Characters[​](http://docs.groupez.dev/zmenu/configurations/informations#2-missing-quotes-for-special-characters "Direct link to 2. Missing Quotes for Special Characters")

`# Wrong - will cause parsing errorname: &6My Menu# Correctname: "&6My Menu"`

### 3. Tab Characters[​](http://docs.groupez.dev/zmenu/configurations/informations#3-tab-characters "Direct link to 3. Tab Characters")

YAML doesn't allow tabs. Always use spaces:

`# Wrong (tabs)items:→ my-button:→ → slot: 0# Correct (spaces)items:  my-button:    slot: 0`

### 4. Duplicate Keys[​](http://docs.groupez.dev/zmenu/configurations/informations#4-duplicate-keys "Direct link to 4. Duplicate Keys")

`# Wrong - second 'slot' overwrites firstmy-button:  slot: 0  slot: 1# Correctmy-button:  slot: 0another-button:  slot: 1`

File Encoding[​](http://docs.groupez.dev/zmenu/configurations/informations#file-encoding "Direct link to File Encoding")
------------------------------------------------------------------------------------------------------------------------

Always save your files with **UTF-8 encoding** to properly support:

*   Special characters
*   Non-English text
*   Color codes

Most modern text editors use UTF-8 by default, but check your editor's settings if you experience issues with special characters.

Creating an Inventory
---------------------

This guide will walk you through creating your first zMenu inventory from scratch.

Step 1: Create the File[​](http://docs.groupez.dev/zmenu/configurations/inventories/create-inventory#step-1-create-the-file "Direct link to Step 1: Create the File")
---------------------------------------------------------------------------------------------------------------------------------------------------------------------

Create a new file in the `plugins/zMenu/inventories/` folder. Name it something descriptive like `my_first_menu.yml`.

`plugins/zMenu/inventories/my_first_menu.yml`

Use lowercase letters, numbers, and underscores for file names. Avoid spaces and special characters.

Step 2: Define Basic Properties[​](http://docs.groupez.dev/zmenu/configurations/inventories/create-inventory#step-2-define-basic-properties "Direct link to Step 2: Define Basic Properties")
---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Start with the essential inventory properties:

`# The title shown at the top of the inventoryname: "&6&lMy First Menu"# The size of the inventory (must be 9, 18, 27, 36, 45, or 54)size: 27# Whether this inventory is enabledenable: true`

Step 3: Add a Fill Item (Optional)[​](http://docs.groupez.dev/zmenu/configurations/inventories/create-inventory#step-3-add-a-fill-item-optional "Direct link to Step 3: Add a Fill Item (Optional)")
----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Fill empty slots with a decorative item:

`name: "&6&lMy First Menu"size: 27enable: truefillItem:  material: GRAY_STAINED_GLASS_PANE  name: "&8"`

This makes your inventory look cleaner by filling empty spaces.

Step 4: Add Your First Button[​](http://docs.groupez.dev/zmenu/configurations/inventories/create-inventory#step-4-add-your-first-button "Direct link to Step 4: Add Your First Button")
---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Now let's add an interactive button. Buttons are defined in the `items` section:

`name: "&6&lMy First Menu"size: 27enable: truefillItem:  material: GRAY_STAINED_GLASS_PANE  name: "&8"items:  welcome-button:    slot: 13    item:      material: DIAMOND      name: "&b&lWelcome!"      lore:        - "&7Click me to see a message"    actions:      - type: message        messages:          - "&aHello, %player%!"          - "&7Thanks for using zMenu!"`

Let's break down what each part does:

| Property | Description |
| --- | --- |
| `welcome_button` | A unique name for this button |
| `slot: 13` | Position in the inventory (center of row 2) |
| `item` | The visual appearance of the button |
| `material` | The Minecraft item type |
| `name` | The item's display name |
| `lore` | Description lines below the name |
| `actions` | What happens when clicked |

Step 5: Test Your Inventory[​](http://docs.groupez.dev/zmenu/configurations/inventories/create-inventory#step-5-test-your-inventory "Direct link to Step 5: Test Your Inventory")
---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

1.   Save the file
2.   Run `/zm reload inventory my_first_menu`
3.   Open it with `/zm open my_first_menu`

You should see a 27-slot inventory with a diamond in the center!

Step 6: Add More Buttons[​](http://docs.groupez.dev/zmenu/configurations/inventories/create-inventory#step-6-add-more-buttons "Direct link to Step 6: Add More Buttons")
------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Let's expand the menu with more functionality:

`name: "&6&lMy First Menu"size: 27enable: truefillItem:  material: GRAY_STAINED_GLASS_PANE  name: "&8"items:  # Title item (non-interactive)  title:    slot: 4    item:      material: BOOK      name: "&6&lServer Guide"      lore:        - "&7Welcome to our server!"        - ""        - "&7Select an option below."  # Teleport to spawn  spawn:    slot: 11    item:      material: RED_BED      name: "&c&lSpawn"      lore:        - "&7Teleport to spawn point"        - ""        - "&eClick to teleport!"    actions:      - type: close      - type: player-command        commands:          - "spawn"  # View player info  info:    slot: 13    item:      material: PLAYER_HEAD      playerHead: "%player%"      name: "&a&l%player%"      lore:        - "&8&m─────────────"        - ""        - "&7Health: &c%player_health%"        - "&7Level: &a%player_level%"        - ""        - "&8&m─────────────"  # Open shop  shop:    slot: 15    item:      material: GOLD_INGOT      name: "&e&lShop"      lore:        - "&7Browse our shop"        - ""        - "&eClick to open!"    actions:      - type: inventory        inventory: "shop"  # Close button  close:    slot: 22    item:      material: BARRIER      name: "&c&lClose"      lore:        - "&7Close this menu"    actions:      - type: close`

Step 7: Add Sound Effects[​](http://docs.groupez.dev/zmenu/configurations/inventories/create-inventory#step-7-add-sound-effects "Direct link to Step 7: Add Sound Effects")
---------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Make the menu more interactive with sounds:

`items:  spawn:    slot: 11    item:      material: RED_BED      name: "&c&lSpawn"    sound: ENTITY_ENDERMAN_TELEPORT    actions:      - type: close      - type: player-command        commands:          - "spawn"`

Or add a sound in the actions:

`actions:  - type: sound    sound: UI_BUTTON_CLICK    pitch: 1.2    volume: 0.5  - type: player-command    commands:      - "spawn"`

Step 8: Add Requirements (Optional)[​](http://docs.groupez.dev/zmenu/configurations/inventories/create-inventory#step-8-add-requirements-optional "Direct link to Step 8: Add Requirements (Optional)")
-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Restrict who can click a button:

`items:  vip-area:    slot: 16    item:      material: DIAMOND_BLOCK      name: "&b&lVIP Area"      lore:        - "&7Exclusive VIP teleport"    click-requirement:      requirements:        - type: permission          permission: "server.vip"          deny:            - type: message              messages:                - "&cYou need VIP rank to use this!"            - type: sound              sound: ENTITY_VILLAGER_NO      success:        - type: player-command          commands:            - "warp vip"`

Understanding Slot Numbers[​](http://docs.groupez.dev/zmenu/configurations/inventories/create-inventory#understanding-slot-numbers "Direct link to Understanding Slot Numbers")
-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Slots are numbered from 0 to (size - 1), starting from the top-left:

`Row 1:  0  1  2  3  4  5  6  7  8Row 2:  9 10 11 12 13 14 15 16 17Row 3: 18 19 20 21 22 23 24 25 26Row 4: 27 28 29 30 31 32 33 34 35Row 5: 36 37 38 39 40 41 42 43 44Row 6: 45 46 47 48 49 50 51 52 53`

For a 27-slot inventory (3 rows), slots 0-26 are available.

Using Multiple Slots[​](http://docs.groupez.dev/zmenu/configurations/inventories/create-inventory#using-multiple-slots "Direct link to Using Multiple Slots")
-------------------------------------------------------------------------------------------------------------------------------------------------------------

A button can occupy multiple slots:

`items:  big-button:    slots:      - 11      - 12      - 13      - 20      - 21      - 22    item:      material: EMERALD_BLOCK      name: "&a&lBig Button"`

Or use a range:

`items:  row-button:    slots:      - 10-16  # Slots 10 through 16    item:      material: GOLD_BLOCK      name: "&e&lGold Row"`

Tips for Good Menu Design[​](http://docs.groupez.dev/zmenu/configurations/inventories/create-inventory#tips-for-good-menu-design "Direct link to Tips for Good Menu Design")
----------------------------------------------------------------------------------------------------------------------------------------------------------------------------

1.   **Use consistent styling** - Keep colors and formatting similar across your menus
2.   **Add visual borders** - Use glass panes or other items to frame your content
3.   **Provide feedback** - Use sounds and messages to confirm actions
4.   **Group related items** - Place similar options near each other
5.   **Include navigation** - Always provide a way to go back or close
6.   **Use clear icons** - Choose materials that visually represent their function
7.   **Keep lore concise** - Don't overload buttons with too much text

Installing zMenu
----------------

This guide will walk you through the process of installing zMenu on your Minecraft server.

Requirements[​](http://docs.groupez.dev/zmenu/installation#requirements "Direct link to Requirements")
------------------------------------------------------------------------------------------------------

Before installing zMenu, make sure your server meets the following requirements:

| Requirement | Version |
| --- | --- |
| Minecraft | 1.19 to 1.21+ |
| Java | **Java 21** (required) |
| Server Software | Spigot, Paper, Purpur, Pufferfish, or Folia |

For the best experience, we recommend using **Paper** or **Purpur**. This enables MiniMessage formatting support and provides better performance.

Download[​](http://docs.groupez.dev/zmenu/installation#download "Direct link to Download")
------------------------------------------------------------------------------------------

You can download zMenu from the following sources:

*   **Modrinth** (Primary): [https://modrinth.com/plugin/zmenu](https://modrinth.com/plugin/zmenu)
*   **SpigotMC**: [https://www.spigotmc.org/resources/zmenu.110402/](https://www.spigotmc.org/resources/zmenu.110402/)

Get the latest development versions in the `#builds` channel on [Discord](https://discord.groupez.dev/).

Installation Steps[​](http://docs.groupez.dev/zmenu/installation#installation-steps "Direct link to Installation Steps")
------------------------------------------------------------------------------------------------------------------------

### Step 1: Download the Plugin[​](http://docs.groupez.dev/zmenu/installation#step-1-download-the-plugin "Direct link to Step 1: Download the Plugin")

Download the latest version of `zMenu.jar` from one of the sources listed above.

### Step 2: Install the Plugin[​](http://docs.groupez.dev/zmenu/installation#step-2-install-the-plugin "Direct link to Step 2: Install the Plugin")

1.   Stop your Minecraft server if it's running
2.   Place the `zMenu.jar` file in your server's `plugins/` folder
3.   Start your server

### Step 3: Verify Installation[​](http://docs.groupez.dev/zmenu/installation#step-3-verify-installation "Direct link to Step 3: Verify Installation")

After starting your server, verify that zMenu loaded correctly:

1.   Check the server console for:

`[zMenu] zMenu has been enabled!` 
2.   Run the command `/zm version` in-game or from console to see version information

### Step 4: Initial Configuration[​](http://docs.groupez.dev/zmenu/installation#step-4-initial-configuration "Direct link to Step 4: Initial Configuration")

After the first startup, zMenu will create the following folder structure:

`plugins/zMenu/├── config.yml                    # Main configuration file├── global-placeholders.yml       # Global placeholder values├── commands/│   └── commands.yml              # Custom command definitions├── inventories/│   ├── basic_inventory.yml       # Example inventory│   ├── advanced_inventory.yml    # Advanced example│   └── pro_inventory.yml         # Pro example├── patterns/│   └── pattern_example.yml       # Example pattern├── items/│   └── default-items.yml         # Reusable item definitions├── actions_patterns/│   └── default-actions.yml       # Default action patterns└── dialogs/                      # Dialog templates (1.20.5+)`

INPUT Button Type
-----------------

This button type requires [zMenu+](https://docs.groupez.dev/zmenu/zmenu-plus) to work.

The `INPUT` button type allows you to capture text input from players. When clicked, it closes the inventory and prompts the player to type in chat. The input can then be used in actions or stored for later use.

Usage[​](http://docs.groupez.dev/zmenu/configurations/buttons/types/input#usage "Direct link to Usage")
-------------------------------------------------------------------------------------------------------

`items:  input-button:    type: INPUT    slot: 13    input-message: "&eEnter your nickname:"    input-cancel: "cancel"    item:      material: NAME_TAG      name: "&a&lChange Nickname"`

Configuration[​](http://docs.groupez.dev/zmenu/configurations/buttons/types/input#configuration "Direct link to Configuration")
-------------------------------------------------------------------------------------------------------------------------------

| Property | Description | Required |
| --- | --- | --- |
| `type` | Must be `INPUT` | Yes |
| `slot` | Button position | Yes |
| `input-message` | Message shown to player when waiting for input | Yes |
| `input-cancel` | Word to type to cancel (default: "cancel") | No |
| `item` | Visual appearance | Yes |

Example[​](http://docs.groupez.dev/zmenu/configurations/buttons/types/input#example "Direct link to Example")

Dialogues
---------

Dialogues are interactive popup windows that appear on screen, allowing players to interact with forms, confirmations, and action buttons. They provide a modern UI experience beyond traditional inventory menus.

Dialogues require **Paper 1.21.7+**.

File Location[​](http://docs.groupez.dev/zmenu/configurations/dialogues#file-location "Direct link to File Location")
---------------------------------------------------------------------------------------------------------------------

Dialogues are stored in the `plugins/zMenu/dialogs/` folder. Each YAML file represents one dialogue.

`plugins/zMenu/dialogs/├── welcome.yml├── confirmation.yml├── server-links.yml└── feedback-form.yml`

Dialog Types[​](http://docs.groupez.dev/zmenu/configurations/dialogues#dialog-types "Direct link to Dialog Types")
------------------------------------------------------------------------------------------------------------------

zMenu supports four dialog types:

| Type | Description |
| --- | --- |
| `notice` | Simple informational dialog with a body and optional inputs |
| `confirmation` | Yes/No confirmation dialog |
| `multi_action` | Multiple action buttons in a grid layout |
| `server_links` | Display server links as clickable buttons |

Basic Structure[​](http://docs.groupez.dev/zmenu/configurations/dialogues#basic-structure "Direct link to Basic Structure")
---------------------------------------------------------------------------------------------------------------------------

`name: "&6&lDialog Title"external_title: "Window Title"type: noticecan-close-with-escape: truepause: falseafter_action: CLOSEbody:  # Body content sections...inputs:  # Input fields...`

Configuration Options[​](http://docs.groupez.dev/zmenu/configurations/dialogues#configuration-options "Direct link to Configuration Options")
---------------------------------------------------------------------------------------------------------------------------------------------

### name[​](http://docs.groupez.dev/zmenu/configurations/dialogues#name "Direct link to name")

The internal name of the dialog, displayed with color formatting.

`name: "&6&lWelcome Dialog"`

### external_title[​](http://docs.groupez.dev/zmenu/configurations/dialogues#external_title "Direct link to external_title")

The window title shown in the dialog header.

`external_title: "Server Information"`

### type[​](http://docs.groupez.dev/zmenu/configurations/dialogues#type "Direct link to type")

The dialog type. See [Dialog Types](http://docs.groupez.dev/zmenu/configurations/dialogues#dialog-types) for details.

`type: notice`

### can-close-with-escape[​](http://docs.groupez.dev/zmenu/configurations/dialogues#can-close-with-escape "Direct link to can-close-with-escape")

Allow players to close the dialog by pressing Escape.

`can-close-with-escape: true`

### pause[​](http://docs.groupez.dev/zmenu/configurations/dialogues#pause "Direct link to pause")

Pause the game while the dialog is open (singleplayer-like behavior).

`pause: false`

### after_action[​](http://docs.groupez.dev/zmenu/configurations/dialogues#after_action "Direct link to after_action")

What happens after an action is performed.

| Value | Description |
| --- | --- |
| `CLOSE` | Close the dialog |
| `PAUSE` | Keep dialog open |
| `NONE` | Do nothing |

`after_action: CLOSE`

* * *

Body Content[​](http://docs.groupez.dev/zmenu/configurations/dialogues#body-content "Direct link to Body Content")
------------------------------------------------------------------------------------------------------------------

The `body` section defines what content appears in the dialog. You can combine multiple content types.

### plain_message[​](http://docs.groupez.dev/zmenu/configurations/dialogues#plain_message "Direct link to plain_message")

Display text messages.

`body:  welcome:    type: plain_message    messages:      - "&6&lWelcome to our server!"      - ""      - "&7Server IP: &fplay.example.com"      - "&7Discord: &fdiscord.gg/example"    width: 400`

### item[​](http://docs.groupez.dev/zmenu/configurations/dialogues#item "Direct link to item")

Display an item with optional tooltip.

`body:  featured-item:    type: item    item:      material: DIAMOND_SWORD      name: "&b&lLegendary Sword"      lore:        - "&7A powerful weapon"    show-decoration: true    show-tooltip: true    width: 200    height: 150`

Input Fields[​](http://docs.groupez.dev/zmenu/configurations/dialogues#input-fields "Direct link to Input Fields")
------------------------------------------------------------------------------------------------------------------

Dialogues can include interactive input fields in the `inputs` section.

### dialog_text[​](http://docs.groupez.dev/zmenu/configurations/dialogues#dialog_text "Direct link to dialog_text")

A text input field.

`inputs:  feedback:    type: dialog_text    label: "&6Share your thoughts:"    width: 400    max-length: 200    multiline:      max-lines: 4`

**Options:**

| Option | Type | Description |
| --- | --- | --- |
| `label` | String | Label displayed above the input |
| `width` | Integer | Width of the input field |
| `max-length` | Integer | Maximum character count |
| `multiline.max-lines` | Integer | Number of lines for multiline input |

### dialog_boolean[​](http://docs.groupez.dev/zmenu/configurations/dialogues#dialog_boolean "Direct link to dialog_boolean")

A Yes/No toggle.

`inputs:  subscribe:    type: dialog_boolean    label: "&eWould you like updates?"    text-true: "&a&lYES"    text-false: "&c&lNo"`

### dialog_single_option[​](http://docs.groupez.dev/zmenu/configurations/dialogues#dialog_single_option "Direct link to dialog_single_option")

A single selection from multiple options.

`inputs:  gamemode:    type: dialog_single_option    label: "&6Select your gamemode:"    options:      survival:        id: "survival"        display: "&2Survival"      creative:        id: "creative"        display: "&bCreative"      adventure:        id: "adventure"        display: "&6Adventure"`

### dialog_number_range[​](http://docs.groupez.dev/zmenu/configurations/dialogues#dialog_number_range "Direct link to dialog_number_range")

`inputs:  amount:    type: dialog_number_range    label: "&eSelect amount:"    start: 1    end: 64    step: 1`

Patterns
--------

zMenu has three types of patterns, each solving a different reuse problem:

| Type | Folder | Purpose | Referenced via |
| --- | --- | --- | --- |
| **Inventory Pattern** | `patterns/` | Reusable decoration / layout buttons shared across inventories | `patterns: ["name"]` on the inventory |
| **Button Pattern** | `patterns/` | Reusable single-button template with variable substitution | `pattern:` section inside an item |
| **Action Pattern** | `actions_patterns/` | Default actions applied to every button in an inventory | `action-patterns: ["name"]` on the inventory |

The first two live in the same folder but are distinguished by the `type` key in the file.

File Locations[​](http://docs.groupez.dev/zmenu/configurations/patterns#file-locations "Direct link to File Locations")
-----------------------------------------------------------------------------------------------------------------------

`plugins/zMenu/├── patterns/               # Inventory patterns and button patterns│   ├── my_decoration.yml   # type: INVENTORY (default)│   └── my_button.yml       # type: BUTTON└── actions_patterns/       # Action patterns    └── default-actions.yml`

Subfolders are supported - `patterns/borders/simple.yml` works.

* * *

Inventory Patterns[​](http://docs.groupez.dev/zmenu/configurations/patterns#inventory-patterns "Direct link to Inventory Patterns")
-----------------------------------------------------------------------------------------------------------------------------------

An inventory pattern defines a set of buttons (decorations, navigation, etc.) that can be applied to any inventory. Think of it as a reusable layer.

### Structure[​](http://docs.groupez.dev/zmenu/configurations/patterns#structure "Direct link to Structure")

`name: "pattern_example"        # Unique name used to reference this patternsize: 54                       # Inventory size (must be a multiple of 9)# type: INVENTORY              # Optional - INVENTORY is the default# enableMultiPage: false       # Set to true to allow pagination buttons to work# matrix:                      # Optional slot matrix (same syntax as inventories)items:  example1:    isPermanent: true    item:      material: IRON_INGOT      name: "&fDecoration"    slots:      - 0      - 8      - 45      - 53`

| Key | Required | Description |
| --- | --- | --- |
| `name` | Yes | Unique identifier. Inventories reference this name. |
| `size` | No | Defaults to `54`. Must be a multiple of 9. |
| `type` | No | `INVENTORY` (default) or `BUTTON`. If set to `BUTTON` the file is loaded as a button pattern instead. |
| `enableMultiPage` | No | When `true`, `NEXT` / `PREVIOUS` buttons inside this pattern will work with the inventory's pagination. Defaults to `false`. |
| `matrix` | No | Optional slot matrix, same format as in inventories. |
| `items` | Yes | The buttons to place. Same syntax as the `items` section in an inventory file. |

### Using in an Inventory[​](http://docs.groupez.dev/zmenu/configurations/patterns#using-in-an-inventory "Direct link to Using in an Inventory")

Reference one or more inventory patterns by name in the `patterns` list:

`name: "&6Shop"size: 54patterns:  - "pattern_example"  - "pagination"items:  # your inventory-specific buttons ...`

Patterns are applied in order - buttons from later patterns override earlier ones if they occupy the same slot.

### Example: Decoration[​](http://docs.groupez.dev/zmenu/configurations/patterns#example-decoration "Direct link to Example: Decoration")

File `patterns/pattern_example.yml` (shipped with zMenu):

`name: "pattern_example"size: 54items:  example1:    isPermanent: true    item:      material: IRON_INGOT      name: "&fExample of pattern"    slots:      - 0      - 8      - 45      - 53`

Usage in an inventory:

`name: "&eExample"size: 54patterns:  - "pattern_example"items:  # ...`

### Example: Navigation with Pagination[​](http://docs.groupez.dev/zmenu/configurations/patterns#example-navigation-with-pagination "Direct link to Example: Navigation with Pagination")

File `patterns/pagination.yml`:

`name: "pagination"size: 54enableMultiPage: trueitems:  next:    type: NEXT    slot: 52    is-permanent: true    item:      material: PAPER      name: "&fNext Page"  previous:    type: PREVIOUS    slot: 49    is-permanent: true    item:      material: PAPER      name: "&fPrevious Page"`

The key here is `enableMultiPage: true` - without it, `NEXT` and `PREVIOUS` buttons from the pattern will not function.

* * *

Button Patterns[​](http://docs.groupez.dev/zmenu/configurations/patterns#button-patterns "Direct link to Button Patterns")
--------------------------------------------------------------------------------------------------------------------------

A button pattern is a template for a **single button**. The inventory that uses it passes variables (`%key%` placeholders) which are replaced before the button is loaded. This is the most powerful pattern type.

### Structure[​](http://docs.groupez.dev/zmenu/configurations/patterns#structure-1 "Direct link to Structure")

`name: "pattern_cookies"       # Unique nametype: BUTTON                  # Must be BUTTONbutton:                       # Everything under here becomes the button definition  slot: '%slot%'  item:    material: '%material%'    name: "&a%name%"    lore:      - "&7Click to buy"  # click-requirement, actions, view_requirement, else ... all go here`

| Key | Required | Description |
| --- | --- | --- |
| `name` | Yes | Unique identifier. |
| `type` | Yes | Must be `BUTTON`. If omitted the file is treated as an inventory pattern and `button:` is ignored. |
| `button` | Yes | The button definition. Supports every property a normal button supports (slot, item, actions, click-requirement, view_requirement, else, etc.). |
| `default-values` | No | Fallback values for variables (see [below](http://docs.groupez.dev/zmenu/configurations/patterns#default-values)). |
| `local-placeholders` | No | Additional placeholders loaded from the pattern file (see [below](http://docs.groupez.dev/zmenu/configurations/patterns#local-placeholders)). |

### Using in an Inventory[​](http://docs.groupez.dev/zmenu/configurations/patterns#using-in-an-inventory-1 "Direct link to Using in an Inventory")

Inside the `items` section of an inventory, add a `pattern` key to the button:

`items:  grandma:    pattern:      file-name: "pattern_cookies"   # The pattern's name      slot: 20                       # Passed as %slot%      material: CAKE                 # Passed as %material%      name: "Grand Ma"              # Passed as %name%      key: "grandma"                # Passed as %key%      price: 10                      # Passed as %price%`

Every key under `pattern:` (except `file-name`, `plugin-name`, and their aliases) becomes a variable. The plugin collects all keys, then replaces `%key%` placeholders in the pattern's `button` section before loading.

### Variable Substitution[​](http://docs.groupez.dev/zmenu/configurations/patterns#variable-substitution "Direct link to Variable Substitution")

Any value wrapped in `%...%` inside the pattern file is replaced with the value provided by the inventory. For example, given:

**Pattern file:**

`button:  slot: '%slot%'  item:    material: '%material%'    name: "&a%name%"`

**Inventory usage:**

`pattern:  file-name: "my_pattern"  slot: 13  material: DIAMOND  name: "Shiny"`

**Result:** slot `13`, material `DIAMOND`, name `&aShiny`.

### Variable Modifiers[​](http://docs.groupez.dev/zmenu/configurations/patterns#variable-modifiers "Direct link to Variable Modifiers")

zMenu provides built-in modifiers that transform a variable's value. If a variable is named `key`, these modifiers are available:

| Modifier | Syntax | Example (`key = hello`) | Result |
| --- | --- | --- | --- |
| Uppercase | `%upper_key%` | `%upper_key%` | `HELLO` |
| Lowercase | `%lower_key%` | `%lower_key%` | `hello` |
| Capitalize | `%capitalize_key%` | `%capitalize_key%` | `Hello` |
| Add one | `%add_one_key%` | `%add_one_key%` (key = `5`) | `6` |
| Remove one | `%remove_one_key%` | `%remove_one_key%` (key = `5`) | `4` |

`add_one` and `remove_one` parse the value as an integer. If parsing fails, the placeholder is left unchanged.

These modifiers work on any variable - for example if the inventory passes `level: 3`, then `%remove_one_level%` resolves to `2`.

### Special Keys[​](http://docs.groupez.dev/zmenu/configurations/patterns#special-keys "Direct link to Special Keys")

These keys are **not** passed as variables - they control how the pattern is loaded:

| Key | Aliases | Description |
| --- | --- | --- |
| `file-name` | `fileName`, `file` | Name of the pattern file (without `.yml`) |
| `plugin-name` | `pluginName`, `plugin` | Load the pattern from another plugin's `patterns/` folder |

### default-values[​](http://docs.groupez.dev/zmenu/configurations/patterns#default-values "Direct link to default-values")

You can define fallback values directly in the pattern file. If the inventory does not provide a variable, the default is used instead:

`name: "my_template"type: BUTTONdefault-values:  material: STONE  name: "Default Name"  slot: 0button:  slot: '%slot%'  item:    material: '%material%'    name: "%name%"`

If the inventory only provides `slot: 5`, the button will use material `STONE` and name `Default Name`.

**Priority:** Inventory values take precedence over default values.

### local-placeholders[​](http://docs.groupez.dev/zmenu/configurations/patterns#local-placeholders "Direct link to local-placeholders")

Additional placeholders can be defined in the pattern file under `local-placeholders`. These are loaded **after** inventory values and default values, so they have the lowest priority among pattern-defined values:

`name: "my_template"type: BUTTONlocal-placeholders:  prefix: "&7[Shop] "button:  item:    name: "%prefix%%name%"`

### List Variables[​](http://docs.groupez.dev/zmenu/configurations/patterns#list-variables "Direct link to List Variables")

When a variable's value is a YAML list, zMenu expands each entry into a separate line in contexts like `lore`. This is useful for dynamic reward descriptions:

**Inventory:**

`items:  reward:    pattern:      fileName: "playtime_reward"      rewards:        - '&f50$'        - '&fx4 Iron Ingots'      commands:        - 'eco give %player% 50'        - 'give %player% iron_ingot 4'`

**Pattern file (lore section):**

`lore:  - '&eRewards:'  - '%rewards%'  - ''`

**Result:** The `%rewards%` line is expanded into two lines (`&f50$` and `&fx4 Iron Ingots`), so the lore shows:

`&eRewards:&f50$&fx4 Iron Ingots`

The same expansion applies to `commands` lists used in actions - a `%commands%` placeholder in a `console_command` action's `commands` list will expand into multiple commands.

### view-requirement and else at Item Level[​](http://docs.groupez.dev/zmenu/configurations/patterns#view-requirement-and-else-at-item-level "Direct link to view-requirement and else at Item Level")

You can add `view-requirement` and `else` from the **inventory side**, outside the pattern's `button` section. These are loaded separately and applied on top of the pattern button:

`items:  my_button:    pattern:      file-name: "my_template"      slot: 10      name: "VIP Item"    view-requirement:      requirements:        - type: permission          permission: "vip.access"    else:      item:        material: BARRIER        name: "&cVIP Only"`

This lets you reuse the same pattern but add different visibility conditions per inventory without modifying the pattern file.

Note: The pattern file itself can also contain `view-requirement` and `else` inside its `button` section - both sources are supported.

### Example: Cookie Clicker[​](http://docs.groupez.dev/zmenu/configurations/patterns#example-cookie-clicker "Direct link to Example: Cookie Clicker")

**Pattern file**`patterns/pattern_cookies.yml`:

`name: "pattern_cookies"type: BUTTONbutton:  click-requirement:    purchase:      clicks:        - ALL      requirements:        - type: placeholder          math: true          placeholder: "%zmenu_player_value_cookie%"          value: "%price%+(%price%*%zmenu_player_value_%key%%*1.15)"          action: SUPERIOR_OR_EQUAL      success:        - type: data          action: SUBTRACT          key: 'cookie'          value: '%price%+(%price%*%zmenu_player_value_%key%%*1.15)'          math: true        - type: data          action: ADD          key: '%key%'          value: '1'        - type: inventory          inventory: 'cookies'      deny:        - type: message          messages:            - "&cYou don't have enough cookies"  slot: '%slot%'  item:    material: '%material%'    name: "&a%name% #353535(&7%zmenu_player_value_%key%%#353535)"    lore:      - ""      - "&fPrice&8: &b%zmenu_formatted_math_%price%+(%price%*{zmenu_player_value_%key%}*1.15)% cookies"      - ""      - "&aClick to buy"`

**Inventory file**`inventories/examples/cookies.yml`:

`name: "&8Cookies"size: 36items:  cookie:    slot: 13    item:      material: COOKIE      name: "#77ff77Cookies#353535: &f%zmenu_player_value_cookie%"      lore:        - ""        - "&fCookie per click&8: &b%zmenu_formatted_math_1+({zmenu_player_value_grandma}*2)+({zmenu_player_value_factory}*5)+({zmenu_player_value_mine}*10)%"        - ""        - "&7Click on me !"    actions:      - type: data        action: ADD        key: 'cookie'        value: '1+(%zmenu_player_value_grandma%*2)+(%zmenu_player_value_factory%*5)+(%zmenu_player_value_mine%*10)'        math: true      - type: refresh  grandma:    pattern:      file-name: "pattern_cookies"      plugin-name: "zMenu"      slot: 20      material: CAKE      name: "Grand Ma"      key: "grandma"      price: 10  factory:    pattern:      file-name: "pattern_cookies"      plugin-name: "zMenu"      slot: 22      material: WHEAT      name: "Factory"      key: "factory"      price: 20  mine:    pattern:      file-name: "pattern_cookies"      plugin-name: "zMenu"      slot: 24      material: IRON_PICKAXE      name: "Mine"      key: "mine"      price: 50`

The same `pattern_cookies` template is reused three times with different variables, creating three upgrade buttons from a single pattern.

### Example: Playtime Rewards[​](http://docs.groupez.dev/zmenu/configurations/patterns#example-playtime-rewards "Direct link to Example: Playtime Rewards")

This example shows nested `else` chains and the `%remove_one_level%` modifier. The pattern displays a different item depending on the player's state (already claimed, not enough playtime, must claim previous level, or ready to claim).

**Pattern file**`patterns/playtime_reward.yml` (simplified):

`name: "playtime_reward"type: BUTTONbutton:  updateOnClick: true  page: '%page%'  slot: '%slot%'  # State 1: Already claimed (level >= current)  view-requirement:    requirements:      - type: placeholder        placeholder: "%zmenu_player_value_playtime_level%"        value: '%level%'        action: SUPERIOR_OR_EQUAL  item:    url: '%head%'    name: '%color%Level %level%'    lore:      - '%rewards%'          # List variable - expands into multiple lines      - '&cAlready claimed'  else:    # State 2: Not enough playtime    view-requirement:      requirements:        - type: placeholder          placeholder: "%zmenu_statistic_hours_played%"          value: '%hour%'          action: LOWER    item:      url: '%head%'      name: '%color%Level %level%'      lore:        - '%rewards%'        - '&cNot enough play time'    else:      # State 3: Must claim previous level first      view-requirement:        requirements:          - type: placeholder            placeholder: "%zmenu_player_value_playtime_level%"            value: '%remove_one_level%'    # level minus 1            action: LOWER      item:        url: '%head%'        name: '%color%Level %level%'        lore:          - '%rewards%'          - '&cClaim the previous reward first'      else:        # State 4: Ready to claim        click-requirement:          right_click:            clicks:              - ALL            requirements:              - type: placeholder                placeholder: "%zmenu_player_value_playtime_level%"                value: '%remove_one_level%'                action: EQUAL_TO              - type: placeholder                placeholder: "%zmenu_statistic_hours_played%"                value: '%hour%'                action: SUPERIOR_OR_EQUAL            success:              - type: data                action: SET                key: 'playtime_level'                value: '%level%'              - type: console_command                commands:                  - '%commands%'       # List variable - expands into multiple commands              - type: sound                sound: ENTITY_PLAYER_LEVELUP        item:          url: '%head%'          name: '%color%Level %level%'          lore:            - '%rewards%'            - '&aClick to claim'`

**Inventory usage** (one button shown):

`items:  btn-10:    pattern:      fileName: "playtime_reward"      slot: 10      level: 1      hour: 1      color: "#7f7f7f"      head: "eyJ0ZXh0dXJlcyI6..."      rewards:        - '&f50$'        - '&fx4 Iron Ingots'      commands:        - 'eco give %player% 50'        - 'give %player% iron_ingot 4'`

Key points:

*   `%remove_one_level%` uses the `remove_one` modifier on the `level` variable - if `level` is `3`, this becomes `2`
*   `%rewards%` is a list variable - it expands into two lore lines
*   `%commands%` is a list variable - it expands into two console commands
*   The nested `else` chain creates a state machine with four visual states

Action Patterns[​](http://docs.groupez.dev/zmenu/configurations/patterns#action-patterns "Direct link to Action Patterns")
--------------------------------------------------------------------------------------------------------------------------

Action patterns define default actions that apply to **every button** in an inventory (unless a button defines its own actions).

### Structure[​](http://docs.groupez.dev/zmenu/configurations/patterns#structure-2 "Direct link to Structure")

`name: default-actionsactions:  - type: sound    sound: ENTITY_VILLAGER_YESdeny-actions:  - type: sound    sound: ENTITY_VILLAGER_NO`

| Key | Description |
| --- | --- |
| `name` | Unique identifier for this action pattern |
| `actions` | Actions executed when a button click succeeds |
| `deny-actions` | Actions executed when a requirement denies a click |

### Using in an Inventory[​](http://docs.groupez.dev/zmenu/configurations/patterns#using-in-an-inventory-2 "Direct link to Using in an Inventory")

Reference action patterns by name in the `action-patterns` list:

`name: "&6Shop"size: 54action-patterns:  - "default-actions"items:  # ...`

Like `patterns`, the `action-patterns` list only accepts **strings**. Objects are not supported.

### Example[​](http://docs.groupez.dev/zmenu/configurations/patterns#example "Direct link to Example")

File `actions_patterns/default-actions.yml` (shipped with zMenu):

`name: default-actionsactions:  - type: sound    sound: ENTITY_VILLAGER_YESdeny-actions:  - type: sound    sound: ENTITY_VILLAGER_NO`

With this pattern applied, every button in the inventory will play `ENTITY_VILLAGER_YES` on a successful click and `ENTITY_VILLAGER_NO` on a denied click, unless the button overrides these actions.

* * *

Variable Resolution Order[​](http://docs.groupez.dev/zmenu/configurations/patterns#variable-resolution-order "Direct link to Variable Resolution Order")
--------------------------------------------------------------------------------------------------------------------------------------------------------

When using button patterns, variables are resolved in this order (highest priority first):

1.   **Inventory values** - keys defined under `pattern:` in the inventory file
2.   **default-values** - fallback values defined in the pattern file
3.   **Global placeholders** - plugin-wide placeholders registered via the API
4.   **local-placeholders** - extra placeholders defined in the pattern file

If the same key exists at multiple levels, the highest-priority source wins.

* * *

Cross-Plugin Patterns[​](http://docs.groupez.dev/zmenu/configurations/patterns#cross-plugin-patterns "Direct link to Cross-Plugin Patterns")
--------------------------------------------------------------------------------------------------------------------------------------------

To load a pattern from another plugin's `patterns/` folder, use `plugin-name`:

`items:  my_button:    pattern:      file-name: "shared_template"      plugin-name: "OtherPlugin"      slot: 13`

This looks for the file `plugins/OtherPlugin/patterns/shared_template.yml`.

Accepted aliases: `plugin-name`, `pluginName`, `plugin`.

* * *

Reloading[​](http://docs.groupez.dev/zmenu/configurations/patterns#reloading "Direct link to Reloading")
--------------------------------------------------------------------------------------------------------

`/zm reload`

All patterns (inventory, button, and action) are reloaded when the plugin restarts or when you run the reload command.

* * *

Common Mistakes[​](http://docs.groupez.dev/zmenu/configurations/patterns#common-mistakes "Direct link to Common Mistakes")
--------------------------------------------------------------------------------------------------------------------------

### Objects in the `patterns` list[​](http://docs.groupez.dev/zmenu/configurations/patterns#objects-in-the-patterns-list "Direct link to objects-in-the-patterns-list")

`# WRONG - will be silently ignoredpatterns:  - pattern:      file-name: "my_decoration"# CORRECTpatterns:  - "my_decoration"`

The `patterns` list uses `getStringList()` internally and only accepts strings. To use a button pattern with variables, place the `pattern:` key inside the `items` section on a specific button - not in the top-level `patterns` list.

### Forgetting `type: BUTTON`[​](http://docs.groupez.dev/zmenu/configurations/patterns#forgetting-type-button "Direct link to forgetting-type-button")

`# WRONG - treated as an inventory pattern, "button:" section is ignoredname: "my_template"button:  slot: '%slot%'  item:    material: DIAMOND# CORRECTname: "my_template"type: BUTTONbutton:  slot: '%slot%'  item:    material: DIAMOND`

Without `type: BUTTON`, the file is loaded as an inventory pattern. Since it has no `items` section, it will fail to load.

### Confusing inventory patterns and button patterns[​](http://docs.groupez.dev/zmenu/configurations/patterns#confusing-inventory-patterns-and-button-patterns "Direct link to Confusing inventory patterns and button patterns")

*   **Inventory patterns** are referenced in the top-level `patterns: ["name"]` list. They define a set of buttons. They do **not** support variable substitution.
*   **Button patterns** are referenced via `pattern:` inside a single item definition. They define one button with `%variable%` placeholders.

### Expecting variables in inventory patterns[​](http://docs.groupez.dev/zmenu/configurations/patterns#expecting-variables-in-inventory-patterns "Direct link to Expecting variables in inventory patterns")

Inventory patterns do not perform variable substitution. If you need `%placeholders%`, use a button pattern (`type: BUTTON`) instead.

Player Data
-----------

zMenu includes a built-in player data system that allows you to store and retrieve values specific to each player. This is perfect for tracking progress, creating toggles, storing statistics, and more.

How It Works[​](http://docs.groupez.dev/zmenu/configurations/player-data#how-it-works "Direct link to How It Works")
--------------------------------------------------------------------------------------------------------------------

Player data is stored in a database (SQLite by default, MySQL/MariaDB optional) and persists across server restarts. Each player has their own set of key-value pairs.

Accessing Player Data[​](http://docs.groupez.dev/zmenu/configurations/player-data#accessing-player-data "Direct link to Accessing Player Data")
-----------------------------------------------------------------------------------------------------------------------------------------------

### Placeholder[​](http://docs.groupez.dev/zmenu/configurations/player-data#placeholder "Direct link to Placeholder")

Use the `%zmenu_player_value_<key>%` placeholder:

`item:  name: "&6Coins: %zmenu_player_value_coins%"  lore:    - "&7Kills: %zmenu_player_value_kills%"    - "&7Deaths: %zmenu_player_value_deaths%"`

### Commands[​](http://docs.groupez.dev/zmenu/configurations/player-data#commands "Direct link to Commands")

`# Set a value/zm players set <player> <key> <value># Add to a value (numeric)/zm players add <player> <key> <value># Get a value/zm players get <player> <key># Remove a key/zm players remove <player> <key># Remove a key from ALL players/zm players removeall <key># List all keys for a player/zm players keys <player>`

Modifying Data with Actions[​](http://docs.groupez.dev/zmenu/configurations/player-data#modifying-data-with-actions "Direct link to Modifying Data with Actions")
-----------------------------------------------------------------------------------------------------------------------------------------------------------------

### The Data Action[​](http://docs.groupez.dev/zmenu/configurations/player-data#the-data-action "Direct link to The Data Action")

`actions:  - type: data    action: SET      # SET, ADD, SUBTRACT, MULTIPLY, DIVIDE, REMOVE    key: "coins"    value: "100"    math: true       # Enable math expressions`

### Actions[​](http://docs.groupez.dev/zmenu/configurations/player-data#actions "Direct link to Actions")

| Action | Description | Example |
| --- | --- | --- |
| `SET` | Set to a specific value | `value: "100"` |
| `ADD` | Add to current value | `value: "50"` |
| `SUBTRACT` | Subtract from current | `value: "25"` |
| `MULTIPLY` | Multiply current value | `value: "2"` |
| `DIVIDE` | Divide current value | `value: "2"` |
| `REMOVE` | Remove the key entirely | - |

### Math Expressions[​](http://docs.groupez.dev/zmenu/configurations/player-data#math-expressions "Direct link to Math Expressions")

Enable math with `math: true`:

`actions:  - type: data    action: SET    key: "total"    value: "%zmenu_player_value_base%*1.5+100"    math: true`

Examples[​](http://docs.groupez.dev/zmenu/configurations/player-data#examples "Direct link to Examples")
--------------------------------------------------------------------------------------------------------

### Coins/Currency System[​](http://docs.groupez.dev/zmenu/configurations/player-data#coinscurrency-system "Direct link to Coins/Currency System")

`# Display coinsitems:  coin-display:    slot: 4    item:      material: GOLD_NUGGET      name: "&6&lYour Coins"      lore:        - "&7Balance: &e%zmenu_player_value_coins%"  # Add coins button  add-coins:    slot: 11    item:      material: GOLD_INGOT      name: "&a&l+100 Coins"    actions:      - type: data        action: ADD        key: "coins"        value: "100"      - type: message        messages:          - "&a+100 coins!"      - type: refresh  # Spend coins  spend-coins:    slot: 15    item:      material: DIAMOND      name: "&b&lBuy Diamond"      lore:        - "&7Cost: &e50 coins"    click-requirement:      requirements:        - type: placeholder          placeholder: "%zmenu_player_value_coins%"          action: SUPERIOR_OR_EQUAL          value: 50          deny:            - type: message              messages:                - "&cYou need 50 coins!"      success:        - type: data          action: SUBTRACT          key: "coins"          value: "50"        - type: console-command          commands:            - "give %player% diamond 1"        - type: refresh`

Default Values[​](http://docs.groupez.dev/zmenu/configurations/player-data#default-values "Direct link to Default Values")
--------------------------------------------------------------------------------------------------------------------------

If a key doesn't exist, the placeholder returns an empty string. Use default values:

`# In requirements, check for existence- type: placeholder  placeholder: "%zmenu_player_value_coins%"  action: SUPERIOR_OR_EQUAL  value: 0  # Works even if key doesn't exist`

Or set defaults on first access:

`items:  initialize:    slot: 0    view-requirement:      requirements:        - type: placeholder          placeholder: "%zmenu_player_value_initialized%"          action: DIFFERENT          value: "true"    item:      material: AIR    actions:      - type: data        action: SET        key: "coins"        value: "0"      - type: data        action: SET        key: "initialized"        value: "true"      - type: refresh`

Best Practices[​](http://docs.groupez.dev/zmenu/configurations/player-data#best-practices "Direct link to Best Practices")
--------------------------------------------------------------------------------------------------------------------------

1.   **Use descriptive keys**: `daily_reward_claimed` not `drc`
2.   **Initialize defaults**: Set initial values for new players
3.   **Use math carefully**: Ensure values are numeric for math operations
4.   **Clean up old data**: Use `removeall` for unused keys
5.   **Backup your database**: Especially before major changes
6.   **Document your keys**: Keep track of what data you're storing

Requirements
------------

Requirements are conditions that must be met before a button is displayed, clicked, or an inventory is opened. They are the core system for creating dynamic, conditional menus in zMenu.

How Requirements Work[​](http://docs.groupez.dev/zmenu/configurations/requirements#how-requirements-work "Direct link to How Requirements Work")
------------------------------------------------------------------------------------------------------------------------------------------------

A requirement checks a condition (permission, placeholder value, item in inventory, etc.). Based on the result:

*   If **all** requirements are met, the `success` actions are executed
*   If **any** requirement fails, the `deny` actions of that requirement are executed

`click-requirement:  vip-requirement:    click:      - ALL    requirements:      - type: permission        permission: "server.vip"        deny:          - type: message            messages:              - "&cYou need VIP to use this!"    success:      - type: message        messages:          - "&aWelcome, VIP!"`

* * *

Where to Use Requirements[​](http://docs.groupez.dev/zmenu/configurations/requirements#where-to-use-requirements "Direct link to Where to Use Requirements")
------------------------------------------------------------------------------------------------------------------------------------------------------------

Requirements can be used in three contexts:

### view-requirement (Buttons)[​](http://docs.groupez.dev/zmenu/configurations/requirements#view-requirement-buttons "Direct link to view-requirement (Buttons)")

Controls whether a button is **visible**. If the requirements are not met, the button is hidden entirely.

`items:  vip-only:    slot: 0    view-requirement:      requirements:        - type: permission          permission: "server.vip"    item:      material: DIAMOND      name: "&b&lVIP Item"`

### click-requirement (Buttons)[​](http://docs.groupez.dev/zmenu/configurations/requirements#click-requirement-buttons "Direct link to click-requirement (Buttons)")

Controls whether a button can be **clicked**. The button remains visible, but the click is denied if conditions are not met.

`items:  purchase:    slot: 0    click-requirement:      <requirement_name>:        requirements:          - type: placeholder            placeholder: "%vault_eco_balance%"            action: SUPERIOR_OR_EQUAL            value: 100            deny:              - type: message                messages:                  - "&cYou need $100 to purchase this!"              - type: sound                sound: ENTITY_VILLAGER_NO        success:          - type: currency-withdraw            amount: 100          - type: message            messages:              - "&aPurchase successful!"    item:      material: GOLD_INGOT      name: "&e&lPurchase - $100"`

### view-requirement / open-requirement (Inventories)[​](http://docs.groupez.dev/zmenu/configurations/requirements#view-requirement--open-requirement-inventories "Direct link to view-requirement / open-requirement (Inventories)")

Controls whether an **inventory can be opened**. Applied at the inventory level, not on individual buttons. `open-requirement` is an alias for `view-requirement` at the inventory level.

`# In your inventory filename: "&6VIP Menu"size: 54view-requirement:  requirements:    - type: permission      permission: "server.vip"      deny:        - type: message          messages:            - "&cYou need VIP to access this menu!"`

If the requirement is not met, the inventory will not open and the deny actions will execute.

* * *

Requirement Types[​](http://docs.groupez.dev/zmenu/configurations/requirements#requirement-types "Direct link to Requirement Types")
------------------------------------------------------------------------------------------------------------------------------------

### permission[​](http://docs.groupez.dev/zmenu/configurations/requirements#permission "Direct link to permission")

Checks if the player has a specific permission.

`requirements:  - type: permission    permission: "server.admin"`

| Key | Type | Description |
| --- | --- | --- |
| `permission` | String | The permission node to check |

You can use any permission from any permission plugin (LuckPerms, etc.). This is the simplest and most common requirement type.

* * *

### placeholder[​](http://docs.groupez.dev/zmenu/configurations/requirements#placeholder "Direct link to placeholder")

Compares a placeholder value against a target. This is the most versatile requirement type.

#### Numeric comparison[​](http://docs.groupez.dev/zmenu/configurations/requirements#numeric-comparison "Direct link to Numeric comparison")

`requirements:  - type: placeholder    placeholder: "%player_level%"    action: SUPERIOR_OR_EQUAL    value: 10`

| Key | Type | Description |
| --- | --- | --- |
| `placeholder` | String | The placeholder to evaluate |
| `action` | String | The comparison operator |
| `value` | Number/String | The value to compare against |

#### String comparison[​](http://docs.groupez.dev/zmenu/configurations/requirements#string-comparison "Direct link to String comparison")

`requirements:  - type: placeholder    placeholder: "%player_world%"    action: EQUALS_STRING    value: "world_nether"`

#### Comparison Operators[​](http://docs.groupez.dev/zmenu/configurations/requirements#comparison-operators "Direct link to Comparison Operators")

| Operator | Description | Example |
| --- | --- | --- |
| `==` | Equal to (numeric) | `action: EQUAL_TO` |
| `!=` | Not equal to (numeric) | `action: DIFFERENT` |
| `>=` | Greater than or equal | `action: SUPERIOR_OR_EQUAL` |
| `>` | Greater than | `action: SUPERIOR` |
| `<=` | Less than or equal | `action: INFERIOR_OR_EQUAL` |
| `<` | Less than | `action: INFERIOR` |
| `equals_string` | Exact string match | `action: EQUALS_STRING` |
| `equalsIgnoreCase` | Case-insensitive string match | `action: EQUALS_IGNORE_CASE` |

#### Math Expressions[​](http://docs.groupez.dev/zmenu/configurations/requirements#math-expressions "Direct link to Math Expressions")

Enable `math: true` to evaluate mathematical expressions in placeholder values:

`requirements:  - type: placeholder    math: true    placeholder: "%zmenu_player_value_cookie%"    value: "%price%+(%price%*%zmenu_player_value_upgrades%*1.15)"    action: SUPERIOR_OR_EQUAL`

This evaluates the math expression before comparing. Useful for dynamic pricing, scaling costs, or complex conditions.

* * *

### money[​](http://docs.groupez.dev/zmenu/configurations/requirements#money "Direct link to money")

Checks if the player has enough money in their account. Works with [BeastTokens](https://www.spigotmc.org/resources/beasttokens-custom-currency.20806/), [Vault](https://www.spigotmc.org/resources/34315/), [PlayerPoints](https://www.spigotmc.org/resources/80745/), [ElementalTokens](https://builtbybit.com/resources/16707/), [ElementalGems](https://builtbybit.com/resources/14920/), [Level](https://www.minecraft.net/), [Experience](https://www.minecraft.net/), [zEssentials](https://www.spigotmc.org/resources/118014/), [EcoBits](https://www.spigotmc.org/resources/109967/), [CoinsEngine](https://www.spigotmc.org/resources/84121/) and [VotingPlugin](https://www.spigotmc.org/resources/15358/).

Uses [CurrenciesAPI](https://github.com/Traqueur-dev/CurrenciesAPI).

`requirements:  - type: money    amount: 1000`

| Key | Type | Description |
| --- | --- | --- |
| `amount` | Number | The minimum balance required |
| `economy` | String | The currency name (optional) |
| `currency` | String | The economy name (only needed for zEssentials, CoinsEngine and EcoBits) |

**Example with specific currency and economy:**

`requirements:  - type: money    amount: 500    currency: "CoinsEngine"    economy: "gems"`

This requirement only checks the balance. To actually withdraw money on success, use the `currency-withdraw` action in the `success` block.

* * *

### item[​](http://docs.groupez.dev/zmenu/configurations/requirements#item "Direct link to item")

Checks if the player has a specific item in their inventory.

`requirements:  - type: item    item:      material: DIAMOND      amount: 5`

| Key | Type | Description |
| --- | --- | --- |
| `item.material` | String | The Minecraft material name |
| `item.amount` | Number | The minimum quantity required |

You can also check for items with specific properties:

`requirements:  - type: item    item:      material: DIAMOND_SWORD      name: "&6Legendary Blade"      amount: 1`

* * *

### luckperm[​](http://docs.groupez.dev/zmenu/configurations/requirements#luckperm "Direct link to luckperm")

Checks if the player belongs to a specific LuckPerms group. Requires [LuckPerms](https://luckperms.net/).

`requirements:  - type: luckperm    group: vip`

| Key | Type | Description |
| --- | --- | --- |
| `group` | String | The LuckPerms group name |

* * *

### job[​](http://docs.groupez.dev/zmenu/configurations/requirements#job "Direct link to job")

Checks if the player has reached a specific job level. Requires [Jobs Reborn](https://www.spigotmc.org/resources/jobs-reborn.4216/).

`requirements:  - type: job    job: Miner    level: 10`

| Key | Type | Description |
| --- | --- | --- |
| `job` | String | The job name |
| `level` | Number | The minimum level required |

* * *

### regex[​](http://docs.groupez.dev/zmenu/configurations/requirements#regex "Direct link to regex")

Checks if a value matches a regular expression pattern. Commonly used with the `INPUT` button type to validate player input.

`requirements:  - type: regex    input: "%input%"    regex: "^[1-9][0-9]?$|^64$"`

| Key | Type | Description |
| --- | --- | --- |
| `input` | String | The value to test against the regex |
| `regex` | String | The regular expression pattern |

**Example: Validate numeric input between 1 and 64:**

`items:  set-amount:    type: INPUT    slot: 13    input-message:      - "&eEnter the amount to purchase:"      - "&7(1-64)"    input-cancel: "cancel"    item:      material: HOPPER      name: "&6&lSet Amount"    click-requirement:      requirements:        - type: regex          input: "%input%"          regex: "^[1-9][0-9]?$|^64$"          deny:            - type: message              messages:                - "&cPlease enter a number between 1 and 64"      success:        - type: data          key: "amount"          value: "%input%"        - type: refresh`

* * *

### player-name[​](http://docs.groupez.dev/zmenu/configurations/requirements#player-name "Direct link to player-name")

Checks the player's name.

`requirements:  - type: player-name    name: "Notch"`

| Key | Type | Description |
| --- | --- | --- |
| `name` | String | The player name to check |

* * *

### cuboid[​](http://docs.groupez.dev/zmenu/configurations/requirements#cuboid "Direct link to cuboid")

Checks if the player is located inside a cuboid zone. You must define the world and the coordinates of the two opposite corners of the cuboid.

`requirements:  - type: cuboid    cuboids:      - "world,0,60,0,100,120,100"`

The format is `<world name>,<x1>,<y1>,<z1>,<x2>,<y2>,<z2>`.

| Key | Type | Description |
| --- | --- | --- |
| `cuboids` | List of Strings | List of cuboid zones in the format `world,x1,y1,z1,x2,y2,z2` |

You can define multiple cuboids. The requirement passes if the player is inside **any** of them:

`requirements:  - type: cuboid    cuboids:      - "world,0,60,0,100,120,100"      - "world,-50,40,-50,50,100,50"      - "world_nether,10,30,10,50,80,50"`

* * *

### and[​](http://docs.groupez.dev/zmenu/configurations/requirements#and "Direct link to and")

This feature is only available with [zMenu+](https://docs.groupez.dev/zmenu/zmenu-plus).

Combines multiple requirements where **all** must be met. This allows you to group requirements together and nest them with `or` blocks for complex logic.

`requirements:  - type: and    requirements:      - type: permission        permission: "server.vip"      - type: placeholder        placeholder: "%player_level%"        action: SUPERIOR_OR_EQUAL        value: 10`

| Key | Type | Description |
| --- | --- | --- |
| `requirements` | List | The list of sub-requirements that must all pass |

* * *

### or[​](http://docs.groupez.dev/zmenu/configurations/requirements#or "Direct link to or")

This feature is only available with [zMenu+](https://docs.groupez.dev/zmenu/zmenu-plus).

Combines multiple requirements where only a **minimum** number must be met.

`requirements:  - type: or    minimum: 1    random: false    requirements:      - type: permission        permission: "server.vip"      - type: permission        permission: "server.premium"`

| Key | Type | Description |
| --- | --- | --- |
| `minimum` | Number | The minimum number of requirements that must pass |
| `random` | Boolean | Whether the list of requirements should be traversed randomly |
| `requirements` | List | The list of sub-requirements |

* * *

### Combining and + or[​](http://docs.groupez.dev/zmenu/configurations/requirements#combining-and--or "Direct link to Combining and + or")

You can combine `and` and `or` requirements to create complex conditions. For example, the player must have permission A **and** B, plus must have permission C **or** D:

`name: "&8Test And/Or"size: 54items:  test:    slot: 22    view-requirement:      requirements:        - type: and          requirements:            - type: permission              permission: zmenu.test              success:                - type: message                  messages:                    - "&7Test And 1 - &aOK"              deny:                - type: message                  messages:                    - "&7Test And 1 - &cKO"            - type: permission              permission: zmenu.test2              success:                - type: message                  messages:                    - "&7Test And 2 - &aOK"              deny:                - type: message                  messages:                    - "&7Test And 2 - &cKO"        - type: or          minimum: 1          random: true          requirements:            - type: permission              permission: zmenu.test3              success:                - type: message                  messages:                    - "&7Test Or 1 - &aOK"              deny:                - type: message                  messages:                    - "&7Test Or 1 - &cKO"            - type: permission              permission: zmenu.test4              success:                - type: message                  messages:                    - "&7Test Or 2 - &aOK"              deny:                - type: message                  messages:                    - "&7Test Or 2 - &cKO"    item:      material: PAPER      name: "&aTest And/Or"    else:      item:        material: PAPER        name: "&cTest And/Or"`

In this example, the player must have `zmenu.test`**and**`zmenu.test2` (the `and` block), plus must have `zmenu.test3`**or**`zmenu.test4` (the `or` block with `minimum: 1`). Each sub-requirement has its own `success` and `deny` actions for feedback.

* * *

Success and Deny Actions[​](http://docs.groupez.dev/zmenu/configurations/requirements#success-and-deny-actions "Direct link to Success and Deny Actions")
---------------------------------------------------------------------------------------------------------------------------------------------------------

Each requirement can have `deny` actions, and the overall requirement block can have `success` actions.

### deny[​](http://docs.groupez.dev/zmenu/configurations/requirements#deny "Direct link to deny")

Actions executed when **this specific requirement** is not met. Defined inside each individual requirement.

`requirements:  - type: permission    permission: "server.vip"    deny:      - type: message        messages:          - "&cYou need VIP rank!"      - type: sound        sound: ENTITY_VILLAGER_NO`

### success[​](http://docs.groupez.dev/zmenu/configurations/requirements#success "Direct link to success")

Actions executed when **all requirements** are met. Defined at the requirement block level (outside the `requirements` list).

`click-requirement:  requirements:    - type: placeholder      placeholder: "%vault_eco_balance%"      action: SUPERIOR_OR_EQUAL      value: 500      deny:        - type: message          messages:            - "&cYou need $500!"  success:    - type: currency-withdraw      amount: 500    - type: console-command      commands:        - "give %player% diamond_sword 1"    - type: message      messages:        - "&aPurchase successful!"    - type: sound      sound: ENTITY_PLAYER_LEVELUP`

All [action types](https://docs.groupez.dev/zmenu/configurations/buttons/actions) can be used in both `deny` and `success` blocks.

* * *

Multiple Requirements[​](http://docs.groupez.dev/zmenu/configurations/requirements#multiple-requirements "Direct link to Multiple Requirements")
------------------------------------------------------------------------------------------------------------------------------------------------

You can combine multiple requirements. **All** requirements must be met for the condition to pass. If any single requirement fails, its `deny` actions are executed.

`click-requirement:  requirements:    - type: permission      permission: "server.vip"      deny:        - type: message          messages:            - "&cYou need VIP rank!"    - type: placeholder      placeholder: "%vault_eco_balance%"      action: SUPERIOR_OR_EQUAL      value: 1000      deny:        - type: message          messages:            - "&cYou need $1000!"    - type: item      item:        material: DIAMOND        amount: 5      deny:        - type: message          messages:            - "&cYou need 5 diamonds!"  success:    - type: message      messages:        - "&aAll conditions met!"`

In this example, the player must have the VIP permission **AND** $1000 **AND** 5 diamonds. If any condition fails, the corresponding deny message is shown.

* * *

The else System[​](http://docs.groupez.dev/zmenu/configurations/requirements#the-else-system "Direct link to The else System")
------------------------------------------------------------------------------------------------------------------------------

The `else` block works with `view-requirement` to display an alternative button when the requirement is not met. Instead of hiding the button entirely, you can show a different item.

### Basic else[​](http://docs.groupez.dev/zmenu/configurations/requirements#basic-else "Direct link to Basic else")

`items:  vip-button:    slot: 0    view-requirement:      requirements:        - type: permission          permission: "vip.access"    item:      material: DIAMOND_BLOCK      name: "&b&lVIP Content"    else:      item:        material: COAL_BLOCK        name: "&7&lLocked"        lore:          - "&cRequires VIP rank"`

If the player has `vip.access`, they see a diamond block. Otherwise, they see a coal block with a "locked" message.

### Nested else Chains[​](http://docs.groupez.dev/zmenu/configurations/requirements#nested-else-chains "Direct link to Nested else Chains")

You can chain multiple `else` blocks to create a state machine with several visual states. Each `else` can have its own `view-requirement`:

`items:  reward:    slot: 0    # State 1: Already claimed    view-requirement:      requirements:        - type: placeholder          placeholder: "%zmenu_player_value_claimed%"          value: "true"          action: EQUAL_TO    item:      material: MINECART      name: "&a&lReward Claimed"      lore:        - "&7You already claimed this reward"    else:      # State 2: Not enough level      view-requirement:        requirements:          - type: placeholder            placeholder: "%player_level%"            value: "10"            action: LOWER      item:        material: BARRIER        name: "&c&lLocked"        lore:          - "&7Requires level 10"      else:        # State 3: Ready to claim        item:          material: CHEST          name: "&e&lClaim Reward"          lore:            - "&7Click to claim!"        click-requirement:          requirements:            - type: placeholder              placeholder: "%player_level%"              value: "10"              action: SUPERIOR_OR_EQUAL          success:            - type: data              action: SET              key: "claimed"              value: "true"            - type: console-command              commands:                - "give %player% diamond 10"            - type: sound              sound: ENTITY_PLAYER_LEVELUP`

The system evaluates each `view-requirement` from top to bottom:

1.   If the first requirement matches, that button state is shown
2.   If not, it moves to the `else` block and checks its requirement
3.   This continues through the chain
4.   The final `else` without a `view-requirement` acts as the default

* * *

Named Click Requirements[​](http://docs.groupez.dev/zmenu/configurations/requirements#named-click-requirements "Direct link to Named Click Requirements")
---------------------------------------------------------------------------------------------------------------------------------------------------------

Inside `click-requirement`, you can define **named requirement groups** with specific click types. This allows different conditions for different click actions.

`click-requirement:  purchase:    clicks:      - ALL    requirements:      - type: placeholder        placeholder: "%vault_eco_balance%"        action: SUPERIOR_OR_EQUAL        value: 100        deny:          - type: message            messages:              - "&cYou need $100!"    success:      - type: currency-withdraw        amount: 100      - type: message        messages:          - "&aPurchased!"`

| Key | Description |
| --- | --- |
| `clicks` | List of click types this group responds to |
| `requirements` | The conditions to check |
| `success` | Actions if all requirements pass |

**Available click types:**`ALL`, `LEFT`, `RIGHT`, `SHIFT_LEFT`, `SHIFT_RIGHT`, `MIDDLE`, `DROP`, `CONTROL_DROP`

**Example with different actions per click:**

`click-requirement:  buy-one:    clicks:      - LEFT    requirements:      - type: placeholder        placeholder: "%vault_eco_balance%"        action: SUPERIOR_OR_EQUAL        value: 100        deny:          - type: message            messages:              - "&cYou need $100!"    success:      - type: currency-withdraw        amount: 100      - type: console-command        commands:          - "give %player% diamond 1"  buy-stack:    clicks:      - RIGHT    requirements:      - type: placeholder        placeholder: "%vault_eco_balance%"        action: SUPERIOR_OR_EQUAL        value: 6400        deny:          - type: message            messages:              - "&cYou need $6400!"    success:      - type: currency-withdraw        amount: 6400      - type: console-command        commands:          - "give %player% diamond 64"`

`items:  initialize:    slot: 0    view-requirement:      requirements:        - type: placeholder          placeholder: "%zmenu_player_value_initialized%"          action: DIFFERENT          value: "true"    item:      material: AIR    actions:      - type: data        action: SET        key: "coins"        value: "0"      - type: data        action: SET        key: "initialized"        value: "true"      - type: refresh`

NONE Button Type
----------------

The `NONE` type is the default button type. It displays an item and executes actions when clicked, with no special built-in behavior.

Usage[​](http://docs.groupez.dev/zmenu/configurations/buttons/types/none#usage "Direct link to Usage")
------------------------------------------------------------------------------------------------------

`items:  my-button:    type: NONE  # Optional - NONE is the default    slot: 13    item:      material: DIAMOND      name: "&b&lMy Button"    actions:      - type: message        messages:          - "&aYou clicked the button!"`

Since `NONE` is the default, you can omit the `type` property:

`items:  my-button:    slot: 13    item:      material: DIAMOND      name: "&b&lMy Button"`

When to Use[​](http://docs.groupez.dev/zmenu/configurations/buttons/types/none#when-to-use "Direct link to When to Use")
------------------------------------------------------------------------------------------------------------------------

Use `NONE` buttons for:

*   Displaying information
*   Executing commands
*   Opening links
*   Sending messages
*   Playing sounds
*   Any custom action

Item Components
---------------

Starting from Minecraft 1.20.5, items use a new component system. zMenu provides full support for this system through the `components` configuration section.

Components require Minecraft 1.20.5 or higher. Some components are only available in newer versions, indicated by a badge next to each component name.

Basic Structure[​](http://docs.groupez.dev/zmenu/configurations/items/components#basic-structure "Direct link to Basic Structure")
----------------------------------------------------------------------------------------------------------------------------------

`item:  material: DIAMOND_SWORD  components:    custom-name: "&6&lLegendary Sword"    lore:      - "&7A powerful weapon"    enchantments:      sharpness: 5`

* * *

attack-range 1.21.11+[​](http://docs.groupez.dev/zmenu/configurations/items/components#attack-range-12111 "Direct link to attack-range-12111")
----------------------------------------------------------------------------------------------------------------------------------------------

Defines the attack range of an item in blocks.

**Minecraft Wiki:**[attack_range](https://minecraft.wiki/w/Data_component_format#attack_range)

| Key | Type | Default | Range | Description |
| --- | --- | --- | --- | --- |
| `min-reach` | Float | `0` | 0-64 | Minimum reach distance in survival mode |
| `max-reach` | Float | `3` | 0-64 | Maximum reach distance in survival mode |
| `min-creative-reach` | Float | `0` | 0-64 | Minimum reach distance in creative mode |
| `max-creative-reach` | Float | `5` | 0-64 | Maximum reach distance in creative mode |
| `hitbox-margin` | Float | `0.3` | 0-1 | Additional margin for hitbox detection |
| `mob-factor` | Float | `1` | 0-2 | Multiplier applied to mob hitboxes |

`components:  attack-range:    min-reach: 0    max-reach: 5    min-creative-reach: 0    max-creative-reach: 6    hitbox-margin: 0.3    mob-factor: 1.0`

* * *

attribute-modifiers[​](http://docs.groupez.dev/zmenu/configurations/items/components#attribute-modifiers "Direct link to attribute-modifiers")
----------------------------------------------------------------------------------------------------------------------------------------------

Adds attribute modifiers to the item.

**Minecraft Wiki:**[attribute_modifiers](https://minecraft.wiki/w/Data_component_format#attribute_modifiers)

| Key | Type | Required | Description |
| --- | --- | --- | --- |
| `type` | String | Yes | Attribute type (e.g., `generic.attack_damage`) |
| `amount` | Double | Yes | Value of the modifier |
| `operation` | String | Yes | Operation type: `add_value`, `add_multiplied_base`, `add_multiplied_total` |
| `slot` | String | No | Equipment slot: `any`, `mainhand`, `offhand`, `head`, `chest`, `legs`, `feet` |
| `id` | String | No | Unique identifier for the modifier |

`components:  attribute-modifiers:    - type: generic.attack_damage      amount: 10      operation: add_value      slot: mainhand      id: "my_plugin:attack_bonus"    - type: generic.movement_speed      amount: 0.1      operation: add_multiplied_base      slot: feet`

**Available Attribute Types:**

*   `generic.armor` - Armor points
*   `generic.armor_toughness` - Armor toughness
*   `generic.attack_damage` - Attack damage
*   `generic.attack_knockback` - Attack knockback
*   `generic.attack_speed` - Attack speed
*   `generic.flying_speed` - Flying speed
*   `generic.follow_range` - Follow range
*   `generic.knockback_resistance` - Knockback resistance
*   `generic.luck` - Luck
*   `generic.max_absorption` - Max absorption
*   `generic.max_health` - Max health
*   `generic.movement_speed` - Movement speed
*   `generic.scale` - Entity scale
*   `generic.step_height` - Step height

Defines patterns on a banner item.

**Minecraft Wiki:**[banner_patterns](https://minecraft.wiki/w/Data_component_format#banner_patterns)

`components:  banner-patterns:    - pattern: stripe_top      color: red    - pattern: cross      color: blue`

blocks-attacks 1.21.5+[​](http://docs.groupez.dev/zmenu/configurations/items/components#blocks-attacks-1215 "Direct link to blocks-attacks-1215")
-------------------------------------------------------------------------------------------------------------------------------------------------

Configures shield-like blocking behavior for items.

**Minecraft Wiki:**[blocks_attacks](https://minecraft.wiki/w/Data_component_format#blocks_attacks)

| Key | Type | Default | Description |
| --- | --- | --- | --- |
| `block-delay-seconds` | Float | `0` | Delay before blocking becomes active |
| `disable-cooldown-scale` | Float | `1` | Cooldown scale when shield is disabled |
| `block-sound` | String | - | Sound played when blocking |
| `disabled-sound` | String | - | Sound played when shield is disabled |
| `item-damage.threshold` | Float | `0` | Damage threshold before item takes damage |
| `item-damage.base` | Float | `0` | Base item damage |
| `item-damage.factor` | Float | `1.5` | Damage multiplier factor |
| `damage-reductions` | List | `[]` | List of damage reduction rules |

**Damage Reduction Properties:**

| Key | Type | Description |
| --- | --- | --- |

Item Configuration
------------------

Items are the visual elements displayed in your inventory buttons. This page documents all available item configuration options.

Basic Item Structure[​](http://docs.groupez.dev/zmenu/configurations/items/item#basic-item-structure "Direct link to Basic Item Structure")
-------------------------------------------------------------------------------------------------------------------------------------------

`item:  material: DIAMOND  name: "&bMy Item"  lore:    - "&7Line 1"    - "&7Line 2"`

Configuration Options[​](http://docs.groupez.dev/zmenu/configurations/items/item#configuration-options "Direct link to Configuration Options")
----------------------------------------------------------------------------------------------------------------------------------------------

### material[​](http://docs.groupez.dev/zmenu/configurations/items/item#material "Direct link to material")

The material type for the item. zMenu supports vanilla Minecraft materials and many custom item plugins.

`item:  material: DIAMOND_SWORD`

#### Material Loaders[​](http://docs.groupez.dev/zmenu/configurations/items/item#material-loaders "Direct link to Material Loaders")

zMenu supports many material sources through its loader system:

| Plugin | Syntax | Price | Link |
| --- | --- | --- | --- |
| **Bukkit** | `STONE`, `DIAMOND_SWORD` | Built-in | Vanilla Minecraft |
| **PlaceholderAPI** | `%your_placeholder_material%` | Free | [SpigotMC](https://www.spigotmc.org/resources/6245/) |
| **Armor (slot)** | `armor:HEAD`, `armor:CHEST`, `armor:LEGS`, `armor:FEET` | Built-in | Player's equipped armor |
| **zHead** | `zhd:<id>` | Free | [Polymart](https://polymart.org/resource/2070) |
| **MagicCosmetics** | `magic_cosmetics:HAT`, `magic_cosmetics:BAG`, etc. | Paid | [Polymart](https://polymart.org/product/6981/magic-cosmetics-gui-non-animated) |
| **HMCCosmetics** | `hmc_cosmetics:<type>`, `hmc_cosmetics:<type>-<player>` | Paid | [SpigotMC](https://www.spigotmc.org/resources/100107/) |
| **zItems** | `zitems:<id>` | Paid | [SpigotMC](https://www.spigotmc.org/resources/118638/) |
| **HeadDatabase** | `hdb:<id>` | Paid | [SpigotMC](https://www.spigotmc.org/resources/14280/) |
| **Oraxen** | `oraxen:<item name>` | Paid | [SpigotMC](https://www.spigotmc.org/resources/72448/) |
| **ItemsAdder** | `itemsadder:<item name>` | Paid | [SpigotMC](https://www.spigotmc.org/resources/73355/) |
| **SlimeFun** | `slimefun:<item name>` | Free | [GitHub](https://github.com/Slimefun/Slimefun4) |
| **Nova** | `nova:<item/block name>` | Free | [GitHub](https://github.com/xenondevs/Nova) |
| **Base64** | `base64:<item in base64>` | Built-in | Import items as base64 string |
| **PlayerHead** | `playerHead:<player name>`, `playerHead:%player%` | Built-in | Display a player's head |
| **CraftEngine** | `craftengine:<item id>` | Free | [Modrinth](https://modrinth.com/plugin/craftengine) |
| **ExecutableItems** | `ei:<item id>` | Paid | [SpigotMC](https://www.spigotmc.org/resources/83070/) |
| **ExecutableBlocks** | `eb:<block id>` | Paid | [SpigotMC](https://www.spigotmc.org/resources/94696/) |
| **Nexo** | `nexo:<item id>` | Paid | [MCModels](https://mcmodels.net/products/13172/nexo) |

**Examples:**

`# Vanilla materialmaterial: DIAMOND_SWORD# Player headmaterial: "playerHead:%player%"# Custom head from zHeadmaterial: "zhd:123"# ItemsAdder custom itemmaterial: "itemsadder:my_namespace:ruby_gem"# Oraxen custom itemmaterial: "oraxen:amethyst_sword"`

* * *

### name[​](http://docs.groupez.dev/zmenu/configurations/items/item#name "Direct link to name")

The display name of the item.

`item:  material: DIAMOND  name: "&b&lShiny Diamond"`

**Features:**

*   Supports color codes (`&6`, `&#FF5500`)
*   Supports MiniMessage (if enabled)
*   Supports placeholders (`%player%`)

* * *

### translated-name[​](http://docs.groupez.dev/zmenu/configurations/items/item#translated-name "Direct link to translated-name")

Use Minecraft's built-in translation system for item names.

`item:  material: DIAMOND_SWORD  translated-name: "item.minecraft.diamond_sword"`

This displays the item name in the player's client language.

* * *

### center-name[​](http://docs.groupez.dev/zmenu/configurations/items/item#center-name "Direct link to center-name")

Center the item name in the tooltip.

`item:  material: DIAMOND  name: "&6&lRare Item"  center-name: true`

* * *

### lore[​](http://docs.groupez.dev/zmenu/configurations/items/item#lore "Direct link to lore")

The description lines shown below the item name.

`item:  material: DIAMOND_SWORD  name: "&6&lLegendary Sword"  lore:    - "&7A powerful weapon"    - ""    - "&7Damage: &c+50"    - "&7Speed: &a+10%"    - ""    - "&eClick to equip!"`

Each list item is a new line. Empty strings (`""`) create blank lines.

* * *

### lore-type[​](http://docs.groupez.dev/zmenu/configurations/items/item#lore-type "Direct link to lore-type")

Define how the lore should interact with existing item lore. This is especially useful when using material loaders (ItemsAdder, Oraxen, etc.) where the loaded item already has lore.

`item:  material: "itemsadder:my_namespace:custom_sword"  lore:    - "&7Additional information"    - "&7Added by zMenu"  lore-type: APPEND  # Add after existing lore`

**Available Types:**

| Type | Description |
| --- | --- |
| `REPLACE` | Replace the existing lore entirely (default) |
| `APPEND` | Add lines after the existing lore |
| `PREPEND` | Add lines before the existing lore |

**Examples:**

`# Replace existing lore (default behavior)item:  material: "oraxen:custom_item"  lore:    - "&7This replaces all existing lore"  lore-type: REPLACE# Add lore after existingitem:  material: "itemsadder:namespace:item"  lore:    - ""    - "&eClick to use!"  lore-type: APPEND# Add lore before existingitem:  material: "zitems:legendary_sword"  lore:    - "&6&lLEGENDARY"    - ""  lore-type: PREPEND`

* * *

### translated-lore[​](http://docs.groupez.dev/zmenu/configurations/items/item#translated-lore "Direct link to translated-lore")

Use Minecraft's built-in translation system for lore lines.

`item:  material: DIAMOND_SWORD  translated-lore:    - "item.minecraft.diamond_sword.desc"`

* * *

### center-lore[​](http://docs.groupez.dev/zmenu/configurations/items/item#center-lore "Direct link to center-lore")

Center all lore lines in the tooltip.

`item:  material: DIAMOND  lore:    - "&7A rare gem"    - "&7Worth a fortune"  center-lore: true`

* * *

### amount[​](http://docs.groupez.dev/zmenu/configurations/items/item#amount "Direct link to amount")

The stack size of the item (1-64).

`item:  material: DIAMOND  amount: 64`

You can also use placeholders:

`item:  material: DIAMOND  amount: "%zmenu_player_value_coins%"  # Dynamic amount`

* * *

### durability[​](http://docs.groupez.dev/zmenu/configurations/items/item#durability "Direct link to durability")

Set the durability/damage value for tools and armor.

`item:  material: DIAMOND_SWORD  durability: 100  # 100 durability used`

* * *

### model-id[​](http://docs.groupez.dev/zmenu/configurations/items/item#model-id "Direct link to model-id")

Set custom model data for resource pack integration.

`item:  material: DIAMOND  model-id: 12345`

This allows resource packs to display custom models for items.

* * *

### item-model 1.21+[​](http://docs.groupez.dev/zmenu/configurations/items/item#item-model-121 "Direct link to item-model-121")

Set the item model using a namespaced key (Minecraft 1.21+).

`item:  material: DIAMOND_SWORD  item-model: "minecraft:custom/my_sword"`

Or with a custom namespace:

`item:  material: DIAMOND_SWORD  item-model: "mypack:weapons/legendary_blade"`

* * *

### equipped-model 1.21+[​](http://docs.groupez.dev/zmenu/configurations/items/item#equipped-model-121 "Direct link to equipped-model-121")

Set the model displayed when the item is equipped (Minecraft 1.21+).

`item:  material: DIAMOND_CHESTPLATE  equipped-model: "minecraft:custom/my_armor"`

This is used for armor and equipment that changes appearance when worn.

* * *

### glow[​](http://docs.groupez.dev/zmenu/configurations/items/item#glow "Direct link to glow")

Add an enchantment glow effect without showing enchantments.

`item:  material: NETHER_STAR  name: "&6&lSpecial Item"  glow: true`

* * *

### player-head[​](http://docs.groupez.dev/zmenu/configurations/items/item#player-head "Direct link to player-head")

Display a player's head. The material is automatically set to `PLAYER_HEAD`.

`item:  player-head: "%player%"  # Current player's head  name: "&a%player%'s Head"`

You can also use a specific player name:

`item:  player-head: "Notch"`

You don't need to specify `material: PLAYER_HEAD` when using `player-head`. The plugin handles this automatically.

* * *

### url[​](http://docs.groupez.dev/zmenu/configurations/items/item#url "Direct link to url")

Use a custom head texture URL (base64 encoded texture).

`item:  material: PLAYER_HEAD  url: "eyJ0ZXh0dXJlcyI6eyJTS0lOIjp7InVybCI6Imh0dHA6Ly90ZXh0dXJlcy5taW5lY3JhZnQubmV0L3RleHR1cmUv..."  name: "&6Custom Head"`

Get head textures from sites like [minecraft-heads.com](https://minecraft-heads.com/).

* * *

### enchantments[​](http://docs.groupez.dev/zmenu/configurations/items/item#enchantments "Direct link to enchantments")

Add enchantments to the item.

`item:  material: DIAMOND_SWORD  enchantments:    - type: SHARPNESS      level: 5    - type: UNBREAKING      level: 3    - type: FIRE_ASPECT      level: 2`

Common enchantment types:

*   Swords: `SHARPNESS`, `SMITE`, `FIRE_ASPECT`, `KNOCKBACK`
*   Armor: `PROTECTION`, `THORNS`, `UNBREAKING`
*   Tools: `EFFICIENCY`, `FORTUNE`, `SILK_TOUCH`
*   Bows: `POWER`, `PUNCH`, `INFINITY`, `FLAME`

* * *

### flags[​](http://docs.groupez.dev/zmenu/configurations/items/item#flags "Direct link to flags")

Hide certain item attributes using item flags.

`item:  material: DIAMOND_SWORD  enchantments:    - type: SHARPNESS      level: 5  flags:    - HIDE_ENCHANTS    - HIDE_ATTRIBUTES`

**Available Flags:**

| Flag | Description |
| --- | --- |
| `HIDE_ENCHANTS` | Hide enchantment list |
| `HIDE_ATTRIBUTES` | Hide attribute modifiers |
| `HIDE_UNBREAKABLE` | Hide "Unbreakable" tag |
| `HIDE_DESTROYS` | Hide "Can destroy" list |
| `HIDE_PLACED_ON` | Hide "Can be placed on" list |
| `HIDE_POTION_EFFECTS` | Hide potion effects |
| `HIDE_DYE` | Hide leather armor dye color |
| `HIDE_ARMOR_TRIM` | Hide armor trim pattern |

Components 1.20.5+[​](http://docs.groupez.dev/zmenu/configurations/items/item#components-1205 "Direct link to components-1205")

Placeholders
------------

zMenu provides built-in placeholders and supports PlaceholderAPI for dynamic content in your inventories.

Built-in Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#built-in-placeholders "Direct link to Built-in Placeholders")
------------------------------------------------------------------------------------------------------------------------------------------------

These placeholders are available without any additional plugins:

### Player Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#player-placeholders "Direct link to Player Placeholders")

| Placeholder | Description | Example Output |
| --- | --- | --- |
| `%player%` | Player's name | `Notch` |

### Pagination Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#pagination-placeholders "Direct link to Pagination Placeholders")

| Placeholder | Description | Example Output |
| --- | --- | --- |
| `%page%` | Current page number | `1` |
| `%maxPage%` | Maximum page number | `5` |
| `%max-page%` | Maximum page number (alias) | `5` |
| `%zmenu_player_page%` | Current page (PAPI format) | `1` |
| `%zmenu_player_next_page%` | Next page number | `2` |
| `%zmenu_player_previous_page%` | Previous page number | `0` |
| `%zmenu_player_max_page%` | Maximum page (PAPI format) | `5` |

### Inventory History Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#inventory-history-placeholders "Direct link to Inventory History Placeholders")

| Placeholder | Description | Example Output |
| --- | --- | --- |
| `%zmenu_player_previous_inventories%` | Number of previous inventories in history | `3` |

### Math Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#math-placeholders "Direct link to Math Placeholders")

Perform calculations directly in your configuration:

| Placeholder | Description | Example |
| --- | --- | --- |
| `%zmenu_math_<expression>%` | Calculate a math expression | `%zmenu_math_5+5%` → `10` |
| `%zmenu_formatted_math_<expression>%` | Formatted math result | `%zmenu_formatted_math_1000+500%` → `1,500` |

**Supported Operations:**

*   Addition: `+`
*   Subtraction: `-`
*   Multiplication: `*`
*   Division: `/`
*   Parentheses: `()`

**Examples:**

`lore:  - "&7Result: %zmenu_math_10*5%"           # Output: 50  - "&7Balance: %zmenu_formatted_math_%vault_eco_balance%*2%"`

### Player Data Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#player-data-placeholders "Direct link to Player Data Placeholders")

| Placeholder | Description |
| --- | --- |
| `%zmenu_player_value_<key>%` | Get player data value |

**Example:**

`lore:  - "&7Coins: &6%zmenu_player_value_coins%"  - "&7Kills: &c%zmenu_player_value_kills%"`

### Command Argument Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#command-argument-placeholders "Direct link to Command Argument Placeholders")

| Placeholder | Description |
| --- | --- |
| `%zmenu_argument_<key>%` | Get command argument value |

This placeholder retrieves arguments passed through custom commands configured in zMenu.

**Example:**

Command configuration:

`# In commands/commands.ymlcommands:  punish:    command: punish    arguments:      - player    inventory: example_punish`

Using the argument in your inventory:

`items:  player-info:    item:      material: PLAYER_HEAD      playerHead: "%zmenu_argument_player%"      name: "&cPunish %zmenu_argument_player%"      lore:        - "&7Click to punish this player"`

When a player runs `/punish Notch`, the placeholder `%zmenu_argument_player%` will return `Notch`.

### Global Placeholder Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#global-placeholder-placeholders "Direct link to Global Placeholder Placeholders")

| Placeholder | Description |
| --- | --- |
| `%zmenu_global_placeholders_<key>%` | Get global placeholder value |

**Example:**

`# In global-placeholders.ymlserver-name: "My Server"# In inventoryname: "&6%zmenu_global_placeholders_server-name%"`

### Time Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#time-placeholders "Direct link to Time Placeholders")

| Placeholder | Description | Example Output |
| --- | --- | --- |
| `%zmenu_time_unix_timestamp%` | Current Unix timestamp | `1704067200` |
| `%zmenu_time_next_day_unix_timestamp%` | Tomorrow's Unix timestamp | `1704153600` |
| `%zmenu_time_today_start_unix_timestamp%` | Start of today timestamp | `1704067200` |

### Statistic Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#statistic-placeholders "Direct link to Statistic Placeholders")

| Placeholder | Description | Example Output |
| --- | --- | --- |
| `%zmenu_statistic_hours_played%` | Hours played (rounded) | `42` |
| `%zmenu_statistic_time_played%` | Formatted playtime | `1d 18h 30m` |

PlaceholderAPI Integration[​](http://docs.groupez.dev/zmenu/configurations/placeholders#placeholderapi-integration "Direct link to PlaceholderAPI Integration")
---------------------------------------------------------------------------------------------------------------------------------------------------------------

zMenu fully supports PlaceholderAPI. Any PAPI placeholder works in:

*   Inventory titles
*   Item names
*   Item lore
*   Messages
*   Requirements
*   Actions

### Popular PlaceholderAPI Expansions[​](http://docs.groupez.dev/zmenu/configurations/placeholders#popular-placeholderapi-expansions "Direct link to Popular PlaceholderAPI Expansions")

Install expansions with: `/papi ecloud download <name>`

| Expansion | Placeholders | Example |
| --- | --- | --- |
| Player | `%player_name%`, `%player_health%`, `%player_level%` | `%player_name%` → `Notch` |
| Vault | `%vault_eco_balance%`, `%vault_eco_balance_formatted%` | `%vault_eco_balance%` → `1500.00` |
| Statistic | `%statistic_deaths%`, `%statistic_kills%` | `%statistic_deaths%` → `42` |
| Server | `%server_online%`, `%server_max_players%` | `%server_online%` → `50` |
| LuckPerms | `%luckperms_prefix%`, `%luckperms_primary_group_name%` | `%luckperms_prefix%` → `[Admin]` |

### Using PlaceholderAPI[​](http://docs.groupez.dev/zmenu/configurations/placeholders#using-placeholderapi "Direct link to Using PlaceholderAPI")

`items:  profile:    slot: 4    item:      material: PLAYER_HEAD      playerHead: "%player_name%"      name: "&6%player_name%'s Profile"      lore:        - "&8&m─────────────────"        - ""        - "&7Health: &c%player_health%&7/&c%player_max_health%"        - "&7Level: &a%player_level%"        - "&7Balance: &6$%vault_eco_balance_formatted%"        - ""        - "&7Kills: &a%statistic_player_kills%"        - "&7Deaths: &c%statistic_deaths%"        - ""        - "&8&m─────────────────"`

Local Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#local-placeholders "Direct link to Local Placeholders")
---------------------------------------------------------------------------------------------------------------------------------------

Define placeholders specific to an inventory:

`# In your inventory filelocal-placeholders:  price: "100"  item-name: "Special Sword"items:  shop-item:    item:      name: "&6%item-name%"      lore:        - "&7Price: &a$%price%"`

Placeholder Caching[​](http://docs.groupez.dev/zmenu/configurations/placeholders#placeholder-caching "Direct link to Placeholder Caching")
------------------------------------------------------------------------------------------------------------------------------------------

For better performance, you can enable placeholder caching in `config.yml`:

`# Enable PlaceholderAPI cachingenable-cache-placeholder-api: true# Cache duration in ticks (20 ticks = 1 second)cache-placeholder-api: 20`

Caching means placeholders won't update instantly. Only enable this if you have performance issues and understand the implications.

Nested Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#nested-placeholders "Direct link to Nested Placeholders")
------------------------------------------------------------------------------------------------------------------------------------------

You can use placeholders inside other placeholders:

`# Using player data in mathlore:  - "&7Double coins: %zmenu_math_%zmenu_player_value_coins%*2%"`

Requirements with Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#requirements-with-placeholders "Direct link to Requirements with Placeholders")
---------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Use placeholders in requirements to create dynamic conditions:

`click-requirement:  requirements:    - type: placeholder      placeholder: "%vault_eco_balance%"      action: SUPERIOR_OR_EQUAL      value: 100      deny:        - type: message          messages:            - "&cYou need at least $100!"`

Placeholder Formatting[​](http://docs.groupez.dev/zmenu/configurations/placeholders#placeholder-formatting "Direct link to Placeholder Formatting")
---------------------------------------------------------------------------------------------------------------------------------------------------

### Numbers[​](http://docs.groupez.dev/zmenu/configurations/placeholders#numbers "Direct link to Numbers")

For formatted numbers, use the `_formatted` variants when available:

*   `%vault_eco_balance_formatted%` instead of `%vault_eco_balance%`
*   `%zmenu_formatted_math_<expr>%` instead of `%zmenu_math_<expr>%`

### Colors in Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#colors-in-placeholders "Direct link to Colors in Placeholders")

Some placeholders include color codes. To strip them:

`name: "%luckperms_prefix% %player_name%"  # Keeps colors`

Debugging Placeholders[​](http://docs.groupez.dev/zmenu/configurations/placeholders#debugging-placeholders "Direct link to Debugging Placeholders")
---------------------------------------------------------------------------------------------------------------------------------------------------

If placeholders aren't working:

1.   **Check PlaceholderAPI is installed**

`/papi info` 
2.   **Test the placeholder**

`/papi parse me %placeholder_name%` 
3.   **Check the expansion is installed**

`/papi list` 
4.   **Install missing expansions**

`/papi ecloud download <expansion>/papi reload`

Inventory Configuration
-----------------------

This page documents all available configuration options for creating inventories in zMenu.

File Location[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#file-location "Direct link to File Location")
---------------------------------------------------------------------------------------------------------------------------------

Inventories are stored in the `plugins/zMenu/inventories/` folder. Each YAML file represents one inventory. You can organize inventories in subfolders.

`plugins/zMenu/inventories/├── main_menu.yml├── shop.yml├── warps/│   ├── spawn.yml│   └── hub.yml└── admin/    └── admin_menu.yml`

Basic Structure[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#basic-structure "Direct link to Basic Structure")
---------------------------------------------------------------------------------------------------------------------------------------

`# Inventory name displayed in the title barname: "&6My Inventory"# Inventory size (9, 18, 27, 36, 45, or 54)size: 54# Enable or disable this inventoryenable: true# Button definitionsitems:  button-name:    # Button configuration...`

Configuration Options[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#configuration-options "Direct link to Configuration Options")
---------------------------------------------------------------------------------------------------------------------------------------------------------

### name[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#name "Direct link to name")

The title displayed at the top of the inventory.

`name: "&6&lMy Server Menu"`

**Features:**

*   Supports color codes (`&6`, `&#FF5500`)
*   Supports MiniMessage (if enabled)
*   Supports placeholders (`%player%`, `%page%`)

**Pagination placeholders:**

`name: "&6Shop &7(Page %page%/%max-page%)"`

* * *

### size[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#size "Direct link to size")

The number of slots in the inventory. Must be a multiple of 9 (for CHEST type).

| Size | Rows | Description |
| --- | --- | --- |
| 9 | 1 | Single row |
| 18 | 2 | Two rows |
| 27 | 3 | Three rows (small chest) |
| 36 | 4 | Four rows |
| 45 | 5 | Five rows |
| 54 | 6 | Six rows (double chest) |

`size: 54`

* * *

### type[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#type "Direct link to type")

The inventory type. Defines the visual layout of the inventory.

`type: CHEST`

**Available types:**

| Type | Size | Description |
| --- | --- | --- |
| `CHEST` | 9-54 | Standard chest inventory (default) |
| `DISPENSER` | 9 | 3x3 dispenser layout |
| `DROPPER` | 9 | 3x3 dropper layout |
| `FURNACE` | 3 | Furnace layout |
| `WORKBENCH` | 10 | Crafting table layout |
| `ENCHANTING` | 2 | Enchanting table layout |
| `BREWING` | 5 | Brewing stand layout |
| `ANVIL` | 3 | Anvil layout |
| `BEACON` | 1 | Beacon layout |
| `HOPPER` | 5 | Hopper layout (5 slots) |
| `SHULKER_BOX` | 27 | Shulker box layout |
| `BARREL` | 27 | Barrel layout |
| `BLAST_FURNACE` | 3 | Blast furnace layout |
| `LECTERN` | 1 | Lectern layout |
| `SMOKER` | 3 | Smoker layout |
| `LOOM` | 4 | Loom layout |
| `CARTOGRAPHY` | 3 | Cartography table layout |
| `GRINDSTONE` | 3 | Grindstone layout |
| `STONECUTTER` | 2 | Stonecutter layout |
| `SMITHING` | 4 | Smithing table layout |

When using a type other than `CHEST`, the `size` option is ignored as each type has a fixed size.

**Example with hopper:**

`name: "&8Quick Select"type: HOPPERitems:  option1:    slot: 0    item:      material: DIAMOND      name: "&bOption 1"`

* * *

### enable[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#enable "Direct link to enable")

Enable or disable this inventory. Disabled inventories cannot be opened.

`enable: true`

This is useful for temporarily disabling an inventory without deleting the file.

* * *

### update-interval[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#update-interval "Direct link to update-interval")

Automatically refresh the inventory at a regular interval (in milliseconds).

`update-interval: 1000  # Refresh every second (1000ms)`

1000ms = 1 second. Use this for inventories with dynamic placeholders that need frequent updates.

* * *

### title-animation[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#title-animation "Direct link to title-animation")

Animate the inventory title with multiple frames. This creates a dynamic title that cycles through different text values.

This feature **requires**[PacketEvents](https://github.com/retrooper/packetevents) to be installed on your server. Without PacketEvents, title animations will not work.

`title-animation:  titles:    - "&bWelcome to &aMenu!"    - "&eEnjoy &four &9Features!"    - "&cRotating &6Animated &2Title!"  cycles: 5  initial-delay: 1  interval: 2  time-unit: SECONDS  show-items-after-animation: true  item-update-interval: 1`

**Configuration options:**

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `titles` | List | - | List of title frames to cycle through |
| `cycles` | Integer | -1 | Number of animation cycles (-1 for infinite) |
| `initial-delay` | Integer | 0 | Delay before animation starts |
| `interval` | Integer | 1 | Time between each frame |
| `time-unit` | String | SECONDS | Time unit for delays (SECONDS, MILLISECONDS, TICKS) |
| `show-items-after-animation` | Boolean | true | Show items after animation completes |
| `item-update-interval` | Integer | 1 | How often to update items during animation |

**Time units:**

*   `NANOSECONDS`
*   `MICROSECONDS`
*   `MILLISECONDS`
*   `SECONDS`
*   `MINUTES`
*   `HOURS`
*   `DAYS`

**Complete example:**

`name: "&6Loading..."size: 27title-animation:  titles:    - "&6&l⏳ Loading."    - "&6&l⏳ Loading.."    - "&6&l⏳ Loading..."    - "&a&l✓ Welcome!"  cycles: 1  initial-delay: 0  interval: 500  time-unit: MILLISECONDS  show-items-after-animation: trueitems:  content:    slot: 13    item:      material: NETHER_STAR      name: "&eMenu Content"`

* * *

### clear-inventory[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#clear-inventory "Direct link to clear-inventory")

Clear the player's inventory while the menu is open and restore it when closed.

`clear-inventory: true`

**Use cases:**

*   Custom texture packs that overlay the entire screen
*   Preventing item interaction while in menu

* * *

### cancel-item-pickup[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#cancel-item-pickup "Direct link to cancel-item-pickup")

Prevent players from picking up items while the inventory is open.

`cancel-item-pickup: true`

**Use cases:**

*   Preventing accidental item pickup during menu interaction
*   Clean GUI experience without interference

* * *

### target-player-name-placeholder[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#target-player-name-placeholder "Direct link to target-player-name-placeholder")

Define a custom placeholder for the target player's name. Useful when opening inventories for other players.

`target-player-name-placeholder: "%target%"`

**Default:**`%player_name%`

**Example usage:**

`name: "&6Profile of %target%"target-player-name-placeholder: "%target%"items:  info:    slot: 13    item:      material: PLAYER_HEAD      name: "&e%target%"      lore:        - "&7Viewing player profile"`

* * *

### action-patterns[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#action-patterns "Direct link to action-patterns")

Apply action patterns to all buttons in this inventory. Action patterns are reusable sets of actions defined in the `plugins/zMenu/action_patterns/` folder.

`action-patterns:  - "default-actions"  - "click-sound"`

**Action pattern file example** (`plugins/zMenu/action_patterns/default-actions.yml`):

`name: default-actionsactions:  - type: sound    sound: ENTITY_VILLAGER_YESdeny-actions:  - type: sound    sound: ENTITY_VILLAGER_NO`

When applied, all buttons in the inventory will inherit these actions. Button-specific actions are executed after pattern actions.

* * *

### fill-item[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#fill-item "Direct link to fill-item")

Define an item to fill all empty slots.

`fill-item:  material: BLACK_STAINED_GLASS_PANE  name: "&8"`

This automatically fills any slot not occupied by a button.

* * *

### patterns[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#patterns "Direct link to patterns")

Apply reusable patterns to this inventory.

`patterns:  - "navigation_bar"  - "border_pattern"`

Patterns must be defined in the `plugins/zMenu/patterns/` folder. See [Patterns](https://docs.groupez.dev/zmenu/configurations/patterns) for details.

* * *

### matrix[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#matrix "Direct link to matrix")

Define the inventory layout visually using characters.

`matrix:  - "AAAAAAAAA"  - "A       A"  - "A   B   A"  - "A       A"  - "AAAAAAAAA"items:  A:    item:      material: BLACK_STAINED_GLASS_PANE      name: "&8"  B:    item:      material: DIAMOND      name: "&bCenter Item"`

Each character represents an item. Spaces are empty slots.

* * *

### items[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#items "Direct link to items")

The buttons in your inventory. This is where you define all clickable elements.

`items:  my-button:    slot: 0    item:      material: DIAMOND      name: "&bDiamond"    actions:      - type: message        messages:          - "&aYou clicked the diamond!"  another-button:    slot: 8    item:      material: EMERALD      name: "&aEmerald"`

See [Button Configuration](https://docs.groupez.dev/zmenu/configurations/buttons/button) for complete button documentation.

* * *

### view-requirement[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#view-requirement "Direct link to view-requirement")

Requirements that must be met to open this inventory.

`view-requirement:  requirements:    - type: permission      permission: "myserver.vip"      deny:        - type: message          messages:            - "&cYou need VIP to access this menu!"`

If requirements aren't met, the inventory won't open and deny actions will execute.

* * *

### open-requirement[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#open-requirement "Direct link to open-requirement")

Alternative name for view-requirement (same functionality).

`open-requirement:  requirements:    - type: permission      permission: "myserver.premium"`

* * *

### open-with-item[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#open-with-item "Direct link to open-with-item")

Define an item that opens this inventory when held and clicked.

`open-with-item:  item:    material: COMPASS    name: "&6&lMenu"    lore:      - "&7Right-click to open"  actions:    - RIGHT_CLICK    - RIGHT_CLICK_BLOCK  type: full  # full, contains, or starts_with`

**Actions types:**

*   `RIGHT_CLICK` - Right click in air
*   `LEFT_CLICK` - Left click in air
*   `RIGHT_CLICK_BLOCK` - Right click on a block
*   `LEFT_CLICK_BLOCK` - Left click on a block

**Type options:**

*   `full` - Item must match exactly
*   `contains` - Item name must contain the specified name
*   `starts_with` - Item name must start with the specified name

Give players the item with:

`/zm giveopenitem <inventory> <player>`

* * *

### open-actions[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#open-actions "Direct link to open-actions")

Actions to execute when the inventory opens.

`open-actions:  - type: sound    sound: ENTITY_EXPERIENCE_ORB_PICKUP  - type: message    messages:      - "&aWelcome to the menu!"`

* * *

### close-actions[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#close-actions "Direct link to close-actions")

Actions to execute when the inventory closes.

`close-actions:  - type: message    messages:      - "&7Thanks for visiting!"`

* * *

### local-placeholders[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#local-placeholders "Direct link to local-placeholders")

Define inventory-specific placeholders.

`local-placeholders:  server-ip: "play.myserver.com"  discord: "discord.gg/myserver"items:  info:    item:      lore:        - "&7IP: &f%server-ip%"        - "&7Discord: &f%discord%"`

* * *

Multi-Page Inventories[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#multi-page-inventories "Direct link to Multi-Page Inventories")
------------------------------------------------------------------------------------------------------------------------------------------------------------

zMenu automatically creates pagination when buttons have multiple slots or pages defined:

`items:  shop-item:    slots:      - 10-16      - 19-25      - 28-34    item:      # This will create multiple pages if there are more items than slots`

Use `NEXT` and `PREVIOUS` button types for navigation. See [Button Types](https://docs.groupez.dev/zmenu/configurations/buttons/types/next) for details.

Best Practices[​](http://docs.groupez.dev/zmenu/configurations/inventories/inventory#best-practices "Direct link to Best Practices")
------------------------------------------------------------------------------------------------------------------------------------

1.   **Organize your inventories** in subfolders by category
2.   **Use patterns** for repeated elements like borders
3.   **Use local placeholders** for values used multiple times
4.   **Test with `/zm reload inventory <name>`** for quick iteration
5.   **Use meaningful button names** for easier maintenance
6.   **Keep inventory sizes appropriate** - don't use 54 slots if you only need 27

Config.yml
----------

The `config.yml` file is the main configuration file for zMenu. It controls global plugin settings, database configuration, performance options, and more.

File Location[​](http://docs.groupez.dev/zmenu/configurations/config-yml#file-location "Direct link to File Location")
----------------------------------------------------------------------------------------------------------------------

`plugins/zMenu/config.yml`

Configuration Options Explained[​](http://docs.groupez.dev/zmenu/configurations/config-yml#configuration-options-explained "Direct link to Configuration Options Explained")
----------------------------------------------------------------------------------------------------------------------------------------------------------------------------

### Debug Settings[​](http://docs.groupez.dev/zmenu/configurations/config-yml#debug-settings "Direct link to Debug Settings")

#### enable-debug[​](http://docs.groupez.dev/zmenu/configurations/config-yml#enable-debug "Direct link to enable-debug")

Enables detailed debug messages in the console. Useful for troubleshooting issues.

`enable-debug: false`

**When to enable:**

*   Troubleshooting inventory loading issues
*   Debugging placeholder problems
*   Reporting bugs to the developer

* * *

#### enable-debug-time[​](http://docs.groupez.dev/zmenu/configurations/config-yml#enable-debug-time "Direct link to enable-debug-time")

Shows timing information for operations.

`enable-debug-time: false`

* * *

### Storage Settings[​](http://docs.groupez.dev/zmenu/configurations/config-yml#storage-settings "Direct link to Storage Settings")

#### storage-type[​](http://docs.groupez.dev/zmenu/configurations/config-yml#storage-type "Direct link to storage-type")

Determines how player data is stored.

| Type | Description |
| --- | --- |
| `SQLITE` | Local file-based database (default) |
| `MYSQL` | MySQL server |
| `MARIADB` | MariaDB server |
| `NONE` | No persistent storage |

`storage-type: SQLITE`

**Recommendations:**

*   **Single server**: Use `SQLITE`
*   **Network/BungeeCord**: Use `MYSQL` or `MARIADB`
*   **No player data needed**: Use `NONE`

* * *

#### database-configuration[​](http://docs.groupez.dev/zmenu/configurations/config-yml#database-configuration "Direct link to database-configuration")

MySQL/MariaDB connection settings.

`database-configuration:  table-prefix: "zmenu_"  host: "localhost"  port: 3306  user: "minecraft"  password: "secure_password"  database: "minecraft_db"`

| Option | Description |
| --- | --- |
| `table-prefix` | Prefix for database tables |
| `host` | Database server address |
| `port` | Database port (default: 3306) |
| `user` | Database username |
| `password` | Database password |
| `database` | Database name |

* * *

### Formatting Settings[​](http://docs.groupez.dev/zmenu/configurations/config-yml#formatting-settings "Direct link to Formatting Settings")

#### enable-mini-message-format[​](http://docs.groupez.dev/zmenu/configurations/config-yml#enable-mini-message-format "Direct link to enable-mini-message-format")

Enables MiniMessage formatting for text. **Only works on Paper/Purpur/Pufferfish**.

`enable-mini-message-format: true`

When enabled, you can use:

`name: "<gradient:red:blue>Gradient Text</gradient>"lore:  - "<rainbow>Rainbow!</rainbow>"  - "<bold><gold>Bold gold</gold></bold>"`

* * *

### Performance Settings[​](http://docs.groupez.dev/zmenu/configurations/config-yml#performance-settings "Direct link to Performance Settings")

#### enable-cache-item-stack[​](http://docs.groupez.dev/zmenu/configurations/config-yml#enable-cache-item-stack "Direct link to enable-cache-item-stack")

Caches created ItemStacks to improve performance.

`enable-cache-item-stack: true`

**Recommendation:** Keep enabled unless you experience issues.

* * *

#### enable-cache-placeholder-api[​](http://docs.groupez.dev/zmenu/configurations/config-yml#enable-cache-placeholder-api "Direct link to enable-cache-placeholder-api")

Caches PlaceholderAPI results to reduce parsing overhead.

`enable-cache-placeholder-api: false`

Enabling this means placeholders won't update instantly. Use with caution.

* * *

#### cache-placeholder-api[​](http://docs.groupez.dev/zmenu/configurations/config-yml#cache-placeholder-api "Direct link to cache-placeholder-api")

Duration to cache placeholder results (in ticks).

`cache-placeholder-api: 20  # 1 second`

* * *

#### cache-offline-player[​](http://docs.groupez.dev/zmenu/configurations/config-yml#cache-offline-player "Direct link to cache-offline-player")

How long to cache offline player data (seconds).

`cache-offline-player: 300  # 5 minutes`

* * *

### Click Settings[​](http://docs.groupez.dev/zmenu/configurations/config-yml#click-settings "Direct link to Click Settings")

#### enable-cooldown-click[​](http://docs.groupez.dev/zmenu/configurations/config-yml#enable-cooldown-click "Direct link to enable-cooldown-click")

Prevents spam clicking by adding a cooldown between clicks.

`enable-cooldown-click: true`

* * *

#### cooldown-click-milliseconds[​](http://docs.groupez.dev/zmenu/configurations/config-yml#cooldown-click-milliseconds "Direct link to cooldown-click-milliseconds")

The cooldown duration between clicks.

`cooldown-click-milliseconds: 100  # 0.1 seconds`

**Adjust based on your needs:**

*   Lower = More responsive, risk of accidental double-clicks
*   Higher = Safer, might feel sluggish

* * *

### Security Settings[​](http://docs.groupez.dev/zmenu/configurations/config-yml#security-settings "Direct link to Security Settings")

#### enable-anti-dupe[​](http://docs.groupez.dev/zmenu/configurations/config-yml#enable-anti-dupe "Direct link to enable-anti-dupe")

Enables the anti-item-duplication system.

`enable-anti-dupe: true`

This detects and prevents common item duplication exploits involving inventory GUIs.

* * *

#### enable-player-open-inventory-logs[​](http://docs.groupez.dev/zmenu/configurations/config-yml#enable-player-open-inventory-logs "Direct link to enable-player-open-inventory-logs")

Logs when players open zMenu inventories.

`enable-player-open-inventory-logs: true`

Useful for:

*   Monitoring player activity
*   Debugging issues
*   Security auditing

* * *

### Menu Settings[​](http://docs.groupez.dev/zmenu/configurations/config-yml#menu-settings "Direct link to Menu Settings")

#### main-menu[​](http://docs.groupez.dev/zmenu/configurations/config-yml#main-menu "Direct link to main-menu")

The default inventory opened by the `MAIN_MENU` button type and the F key shortcut.

`main-menu: "example"`

This should match an inventory filename (without `.yml`) in your `inventories/` folder.

* * *

#### use-swap-item-off-hand-key-to-open-main-menu[​](http://docs.groupez.dev/zmenu/configurations/config-yml#use-swap-item-off-hand-key-to-open-main-menu "Direct link to use-swap-item-off-hand-key-to-open-main-menu")

Allow players to press F (swap offhand key) to open the main menu.

`use-swap-item-off-hand-key-to-open-main-menu: false`

**When to enable:**

*   You want easy menu access without commands
*   Your server has a main hub menu

**When to keep disabled:**

*   Players need the F key for actual item swapping
*   You use custom resource packs with F key bindings

* * *

Reloading Configuration[​](http://docs.groupez.dev/zmenu/configurations/config-yml#reloading-configuration "Direct link to Reloading Configuration")
----------------------------------------------------------------------------------------------------------------------------------------------------

After making changes:

`/zm reload config`

Or reload everything:

`/zm reload`

Database configuration changes require a server restart to take effect.

Best Practices[​](http://docs.groupez.dev/zmenu/configurations/config-yml#best-practices "Direct link to Best Practices")
-------------------------------------------------------------------------------------------------------------------------

1.   **Start with defaults**: Only change what you need
2.   **Enable caching on large servers**: Improves performance
3.   **Use MySQL for networks**: Allows data sharing across servers
4.   **Keep anti-dupe enabled**: Protects your economy
5.   **Test debug mode**: Useful when setting up, disable in production
6.   **Set appropriate click cooldown**: Balance responsiveness and safety

Troubleshooting[​](http://docs.groupez.dev/zmenu/configurations/config-yml#troubleshooting "Direct link to Troubleshooting")
----------------------------------------------------------------------------------------------------------------------------

### Database Connection Failed[​](http://docs.groupez.dev/zmenu/configurations/config-yml#database-connection-failed "Direct link to Database Connection Failed")

1.   Verify credentials are correct
2.   Check database server is running
3.   Ensure the database exists
4.   Check firewall allows connections
5.   Verify user has proper permissions

### Placeholders Not Updating[​](http://docs.groupez.dev/zmenu/configurations/config-yml#placeholders-not-updating "Direct link to Placeholders Not Updating")

1.   Disable placeholder caching temporarily
2.   Check if PlaceholderAPI is installed
3.   Verify the expansion is downloaded

### Performance Issues[​](http://docs.groupez.dev/zmenu/configurations/config-yml#performance-issues "Direct link to Performance Issues")

1.   Enable item stack caching
2.   Enable placeholder caching
3.   Increase click cooldown
4.   Check for inventory loops (A opens B opens A)
